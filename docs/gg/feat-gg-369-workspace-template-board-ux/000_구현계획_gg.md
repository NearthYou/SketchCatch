# 이슈 #369 Workspace·Template Board UX 구현 계획

## 1. 목표

이 문서는 이슈 #369의 완료 조건을 하나의 구현 흐름으로 고정한다.

- DB schema 작업을 직접 수행한 Codex만 migration 번호 충돌 경고를 즉시·최종 보고한다.
- Template·Project 카드와 큰 미리보기는 임의 축약 그림이 아니라 실제 Architecture Board의 raster 캡처를 사용한다.
- `/workspace/new`의 `빈 보드로 시작`은 한 번의 활성화로 프로젝트 생성과 Workspace 이동을 끝낸다.
- 여섯 deployable Template과 Workspace에서 VPC/Subnet, Security Group, ASG, IGW, Route/NAT의 시각 문법을 동일하게 적용한다.
- 사용자용 label, Terraform local name, AWS `Name` 값을 구분한다.

## 2. 확정된 표현 규칙

### 2.1 실제 포함 관계

- VPC, Availability Zone, Subnet만 기본적인 물리·네트워크 포함 Area로 사용한다.
- Security Group은 AWS의 물리적 컨테이너가 아니므로 drag/drop parent로 저장하지 않는다.
- Security Group을 읽기 쉽게 보여줄 때는 반투명 점선 `보안 범위`로 렌더링하되, 적용 대상 리소스와 명시적인 관계선을 함께 표시한다.
- 하나의 리소스가 여러 Security Group을 사용하는 경우에도 parent를 중복하지 않고 여러 범위/관계로 표현한다.
- ASG는 Terraform Resource node다. 시각적인 묶음이 필요하면 ASG 자체를 Area로 바꾸지 않고 별도의 Presentation Group을 사용한다.

### 2.2 경계 리소스

- Internet Gateway는 VPC 경계에 걸쳐 보이도록 배치한다.
- Route Table과 Route Table Association은 관련 Subnet 경계 가까이에 배치한다.
- NAT Gateway는 실제 소속 Public Subnet 내부에 배치한다.
- 위 배치는 카드, 큰 미리보기, 실제 Workspace에서 같은 좌표 계약을 사용한다.

### 2.3 이름

- Board에는 `Public Subnet A`, `ECS Service` 같은 친화적 label을 표시한다.
- Terraform local name은 영문 소문자·숫자·underscore만 사용하는 짧고 결정적인 내부 식별자로 만든다.
- 정규화 결과가 충돌할 때만 짧은 결정적 suffix를 붙인다.
- 오른쪽 metadata에는 generic `Resource name`이 아니라 `Terraform resource name` 또는 `Terraform address`라고 표시한다.
- AWS `Name` tag/parameter는 Terraform local name과 별도 값으로 유지한다.
- 모든 `@ref`와 Terraform reference는 최종 local name을 통해 다시 계산한다.
