# UI 재구축 전 현재 구조 분석

이 문서는 `refactor/gg/305-ui-ux-rebuild` 브랜치에서 새 UI를 만들기 전에, 현재 코드에서 무엇을 살리고 무엇을 바꿔야 하는지 정리한 Milestone 0 결과다.

한 줄로 말하면:

> 화면은 대부분 placeholder 상태지만, 인증·프로젝트·Architecture Board·Terraform·Reverse Engineering·Deployment 기능은 남아 있다. 새 UI는 이 기능을 다시 연결해야 한다.

## 1. 분석 기준

아래 문서를 처음부터 끝까지 읽고 현재 코드와 비교했다.

- `DESIGN.md`
- `docs/gg/007_UI재구축마일스톤_gg.md`
- 이 대화에 첨부된 외부 화면 구성 handoff 문서
- `apps/web/AGENTS.md`
- `docs/gg/003_기획서.md`
- `packages/types/src/index.ts`
- `docs/data-models.md`

판단 순서는 다음과 같다.

```text
shared type과 canonical 문서
→ DESIGN.md
→ apps/web/AGENTS.md
→ UI 재구축 마일스톤
→ 화면 구성 handoff
```

화면 구성 handoff의 `Architecture Board 자동 생성`은 사용자 승인 원칙과 충돌한다. 새 UI에서는 아래처럼 해석한다.

```text
Architecture Draft 미리보기
→ 사용자가 적용
→ Architecture Board 변경
```

## 2. 현재 기술 구조

| 항목 | 현재 값 | UI 재구축 기준 |
| --- | --- | --- |
| package manager | pnpm 11.8.0 | 기존 workspace와 script 유지 |
| frontend | Next.js 16.2.9, React 19.2.7 | App Router 유지 |
| language | TypeScript 6.0.3 | shared type 우선 |
| Board | `@xyflow/react` 12.11.0 | React Flow 유지 |
| icon | `lucide-react` 1.21.0 | 새 버튼 icon에 우선 사용 |
| style | CSS Modules 일부, global stylesheet 없음 | 새 token layer와 route shell 필요 |
| auth state | `AuthProvider` | 세션 복구와 실제 API 연결 유지 |

현재 `apps/web/app/layout.tsx`는 React Flow CSS와 `AuthProvider`만 연결한다. 공통 global stylesheet는 없다.

`LoginForm`, `SignupForm`, Dashboard client가 사용하는 예전 class 이름은 남아 있지만 이를 받는 공통 CSS는 현재 route에 연결되어 있지 않다. 따라서 예전 UI 모양을 복원하지 않고 `DESIGN.md` 기준의 새 공통 style을 만든다.

Cloudscape, shadcn/ui, Tremor는 현재 dependency가 아니다. 이들은 화면 구조 참고용이며, 새 library를 무조건 설치하지 않는다.

## 10. 위험과 구현 기준

### 가장 큰 위험

1. placeholder를 지우면서 실제 client까지 같이 버리는 것
2. demo dashboard 데이터를 실제 프로젝트처럼 다시 쓰는 것
3. AI 또는 Reverse 결과를 미리보기 없이 Board에 반영하는 것
4. Repository Analysis가 없는데 있는 것처럼 GitHub Repo 흐름을 완성하는 것
5. High Risk가 있는데 Apply CTA를 활성화하는 것
6. Terraform 또는 AWS 작업을 frontend에서 직접 실행하는 것
7. 기존 CSS를 참고해 새 UI를 다시 낡은 화면처럼 만드는 것

### 구현 기준

- Milestone 1부터 `DESIGN.md` token을 쓰는 공통 style layer를 먼저 만든다.
- 기존 global class 이름을 디자인 근거로 사용하지 않는다.
- 실제 기능이 있으면 API를 연결한다.
- 기능이 없으면 disabled 상태와 `아직 연결되지 않음`을 표시한다.
- demo/mock은 별도 파일과 명확한 label을 사용한다.
