# ck 배포 실행 선택지

이 문서는 ck 배포 파트에게 묻는 문서다.

쉽게 말하면:

> Plan/Apply 중 생긴 오류를 gg AI가 어떤 모양으로 받아서 설명할지 정하는 문서다.

## 1. AI 오류 설명에 무엇을 보낼까?

### A. 최소 정보만 보내기. 추천

```ts
{
  stage: "validate" | "export" | "plan" | "apply";
  rawMessage: string;
  relatedResourceId?: string;
}
```

AI는 이 정도만 있어도 쉬운 설명을 만들 수 있다.

로그 전체를 보내면 민감정보가 섞일 수 있다.

ck 배포 파트는 보통 `validate`, `plan`, `apply`를 쓰고, `export`는 gg/sw IaC Preview/export 흐름에서 쓴다.

### B. 배포 id와 로그 전체를 보내기

맥락은 많지만 위험하다.

Secret masking 문제가 커진다.

### C. AI가 deployment DB를 직접 조회

AI가 배포 DB 구조에 묶인다.

피하는 게 좋다.

## 2. stage 값은 어디까지 둘까?

### A. `validate`, `export`, `plan`, `apply`로 맞추기. 추천

Terraform 흐름과 잘 맞는다.

단, ck 배포 로그 자체는 `validate`, `plan`, `apply` 중심으로 남겨도 된다.

### B. `connection_test`, `cleanup`까지 포함

더 자세하지만 gg 범위가 커진다.

### C. 문자열 자유 입력

팀원마다 다른 값을 넣을 수 있어 위험하다.

## 3. AI 설명을 배포 이력에 저장할까?

### A. MVP에서는 저장하지 않음. 추천

ck는 원본 로그와 배포 이력을 저장한다.

gg AI는 요청받을 때 설명만 반환한다.

### B. AI 설명도 저장

나중에 다시 보기 좋다.

하지만 DB schema, 오래된 설명 처리, 보안 문제가 생긴다.

### C. 전부 AI 쪽에서 저장

책임이 섞여서 위험하다.

## 4. 배포 전 점검은 언제 보여줄까?

### A. Plan/Apply 전에 보여주기. 추천

사용자가 실제 배포 전에 위험을 볼 수 있다.

### B. Apply 실패 후 보여주기

이미 실패한 뒤라 예방 가치가 약하다.

### C. 안 보여주기

SketchCatch 핵심 가치가 약해진다.

## ck가 맞춰주면 좋은 형태

아래 두 예시는 ck 배포 결과와 로그용 stage다.
AI 오류 설명 입력 타입과 다르게 `export`를 넣지 않고, 배포 시작 상태인 `init`을 포함한다.

배포 결과 요약 예:

```ts
type PlanApplyOutput = {
  deploymentId: string;
  projectId: string;
  architectureId: string;
  stage: "init" | "validate" | "plan" | "apply";
  status: "PENDING" | "RUNNING" | "SUCCESS" | "FAILED" | "CANCELLED";
  summary?: {
    createCount: number;
    updateCount: number;
    deleteCount: number;
    replaceCount: number;
    blocked: boolean;
  };
};
```

로그 한 줄 예:

```ts
type DeploymentLogLine = {
  deploymentId: string;
  sequence: number;
  stage: "init" | "validate" | "plan" | "apply";
  level: "INFO" | "WARN" | "ERROR";
  message: string;
  relatedResourceId?: string;
};
```

## ck에게 받고 싶은 답

```text
ck 선택 결과

1. AI 오류 설명 입력: A/B/C
2. stage 범위: A/B/C
3. AI 설명 저장: A/B/C
4. 배포 전 점검 시점: A/B/C
```
