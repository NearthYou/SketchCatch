# 기존 AWS 가져오기와 AI 초안 사용성 결정

이 문서는 이슈 #474를 구현하기 전에 확정한 내용을 기록한다.

## 확정된 기준

### 기존 AWS 가져오기

- 가져온 원본 구조를 먼저 보여준다.
- AWS에는 보드 좌표가 없으므로, 발견한 Resource·관계·설정은 그대로 두고 겹치지 않게 위치만 기본 배치한 상태를 `원본`으로 본다.
- 이 기본 배치는 `자동 정리`가 아니며 Compiler가 Resource 의미를 다시 제안한 결과도 아니다.
- 자동 정리는 기본으로 적용하지 않는다.
- 사용자가 원할 때만 정리된 모습을 미리 보여준다.
- 사용자가 직접 선택하기 전에는 원본 구조와 보드를 바꾸지 않는다.
- 같은 실패를 여러 번 세지 않고 실제로 못 읽은 서비스 수만 보여준다.
- 오류 원문 대신 서비스 이름과 해결 방법을 짧고 쉽게 안내한다.
- 닫힌 상세 영역은 `+`, 열린 상세 영역은 `−`로 표시한다.
- 기존 AWS 연결에 가져오기 권한이 부족해도 새 연결을 만들게 하지 않는다.
- `가져오기 권한 추가`로 같은 연결의 권한 갱신을 시작하고, 사용자가 AWS에서 승인한 뒤 같은 연결을 다시 확인한다.
- 권한 갱신 전에는 SketchCatch가 AWS 설정을 임의로 바꾸지 않는다.
- 일부 서비스만 읽지 못해도 성공적으로 가져온 Resource는 원본 보드에 바로 보여준다.
- 일부 조회 실패 때문에 전체 결과 화면을 막지 않는다.
- 상단에는 `일부 항목을 가져오지 못했어요`와 `가져오기 권한 추가`만 간단히 보여준다.
- 일부 항목이 빠진 결과도 사용자가 선택하면 보드에 적용할 수 있다.
- 이때 적용 버튼은 `가져온 항목만 사용`으로 표시해 완전한 결과처럼 보이지 않게 한다.
- `가져온 항목만 사용` 클릭을 최종 승인으로 보고 추가 확인창은 띄우지 않는다.

### AI 초안 만들기

- 선택지를 누르면 선택한 답변과 다음 AI 응답까지 대화가 자연스럽게 따라간다.
- 사용자가 이전 대화를 읽는 동안에는 화면을 강제로 아래로 내리지 않는다.
- 모바일에서도 최신 대화와 입력창에 쉽게 도달할 수 있어야 한다.
- 미리보기 제목은 `초안이 준비됐어요`로 표시한다.
- 기본 설명은 구조를 설명하는 쉬운 한 문장, Resource 수, 연결 수, 확인할 점 최대 3개만 보여준다.
- 확인할 점이 3개를 넘으면 `모두 보기`에서 추가 내용을 확인할 수 있게 하고 안전 정보를 버리지 않는다.
- Compiler, 후보 ID, 내부 상태와 처리 방식은 사용자 화면에 보여주지 않는다.
- 데스크톱에서는 불필요한 `미리보기로 이동` 버튼을 두지 않는다.
- 모바일에서는 미리보기가 완성된 뒤에만 `미리보기 보기` 버튼을 보여준다.
- 데스크톱 대화 중에는 전체 Canvas 위에 큰 채팅 Dock을 띄우는 C안 구조를 사용한다.
- Orbit 중심은 화면 가운데보다 살짝 오른쪽에 두고, 궤도 일부가 채팅 Dock 뒤에 가려지는 것을 허용한다.
- 초안 완성 후에는 채팅을 좁은 왼쪽 rail로 바꾸고 실제 Preview를 넓게 보여주는 A안 구조로 전환한다.
- 평상시 AWS icon 공전 animation을 유지한다.
- option 선택, 직접·확정 음성 답변 전송, assistant 응답 도착 때 Orbit이 짧게 움찔한다.
- 대화 답변이 누적되면 바깥 궤도부터 하나씩 사라지고 Resource icon이 중앙으로 모인다.
- 답변 수는 전체 질문 수나 완성률을 뜻하지 않으며 percentage를 표시하지 않는다.
- Architecture Draft가 준비되고 Compiler가 처리하는 동안 icon이 한 점으로 수렴한다.
- Compiler 성공 뒤에만 점을 없애고 실제 Preview를 공개한다. 장식 icon을 최종 Resource로 재사용하지 않는다.
- 모바일은 채팅을 화면 전체로 보여주고 Orbit은 채팅 뒤에서 일부만 보이게 한다.
- 모바일에서 초안이 완성되면 `미리보기 보기`를 표시하고, 누르면 전체 화면 Preview로 전환하며 `대화로 돌아가기`를 제공한다.

### 보드 정리 미리보기

- `자동 정리`는 Resource, 관계, 설정을 바꾸지 않는다.
- `자동 정리`가 바꾸는 것은 위치, 영역 배치, 연결선 모양 같은 시각적 정리뿐이다.
- 정리 전후는 같은 보드에서 `원본`과 `정리 결과`를 전환하며 비교한다.
- 화면을 둘로 나눠 원본과 정리 결과를 동시에 보여주지 않는다.
- 비교 중에는 어느 쪽을 보고 있는지 분명하게 표시하고, 전환만으로 보드를 변경하지 않는다.
- 선택 버튼은 `원본 유지`와 `이 정리 사용` 두 개만 둔다.
- `원본 유지`는 미리보기를 닫고 아무것도 변경하지 않는다.
- `이 정리 사용` 클릭을 최종 승인으로 보고 정리 결과를 적용하며, 추가 확인창은 띄우지 않는다.
- 점수, 거리, 후보 ID, 내부 버전, 참고 자료 ID를 사용자에게 보여주지 않는다.
- 사용자가 알아야 할 것은 무엇이 달라졌는지, 무엇을 확인해야 하는지, 원본과 정리 결과 중 무엇을 쓸지다.
- 내부 진단 문구는 사용자 화면이 아니라 개발자 확인용 정보로 남긴다.
- 버튼과 설명은 처음 보는 사람도 결과를 선택할 수 있는 쉬운 말로 바꾼다.

## 지켜야 할 경계

- 가져온 원본과 정리된 미리보기를 같은 결과처럼 섞지 않는다.
- 대화 중 장식 화면을 실제 추천 결과처럼 설명하지 않는다.
- 사용자가 직접 적용하기 전에는 프로젝트나 보드를 바꾸지 않는다.

## 그릴링 종료

- 기존 AWS 가져오기, AI 초안 만들기, Board 자동 정리의 제품 경계와 사용자 선택을 모두 확정했다.
- 세 작업은 같은 이슈 #474와 브랜치를 사용하되 담당 파일을 나눠 병렬로 구현한다.
