# gg 문서 구조

이 문서는 gg 담당 작업 문서를 어디서 보면 되는지 정리한 안내 문서다.

쉽게 말하면:

> 계속 보는 큰 문서는 `docs/gg` 바로 아래에 두고, 현재 작업 문서는 해당 브랜치 이름 폴더에 둔다. 5일 넘게 갱신하지 않은 작업 문서는 `old`에 옮기고, 예전 AI 전담 시절 문서는 `old-ai-work`에 모아둔다.

| `old/feat-gg-202-0706-feedback-reverse-template` | Reverse Engineering·Template 피드백 작업 기록 |

## 5. private 문서

`docs/private`에는 GitHub에 올리지 않을 개인 문서를 둔다.

예:

- 발표 스크립트
- 시연 시나리오
- 커밋 코드 설명 스크립트
- 팀원에게 설명할 연결 시나리오
- 면접 준비 답안지
- 아직 공유하기 애매한 개인 메모

공유해야 하는 구현 계획이나 결정 문서는 `docs/private`에 두지 않고 `docs/gg`로 옮긴다.

## 6. private 폴더 규칙

`docs/private` 안에는 별도 `etc` 폴더를 만들지 않는다.
