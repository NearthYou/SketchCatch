# gg 문서 구조

이 문서는 gg AI 문서를 어디서 보면 되는지 정리한 안내 문서다.

쉽게 말하면:

> 계속 보는 큰 문서는 `docs/gg` 바로 아래에 두고, 특정 작업 브랜치에서 만든 문서는 그 브랜치 이름 폴더에 둔다.

## 1. 최상위 문서

`docs/gg` 바로 아래 문서는 전체 흐름을 볼 때 읽는다.

| 문서 | 언제 읽나 |
| --- | --- |
| `000_MVP범위_gg.md` | gg AI가 하는 일과 하지 않는 일을 볼 때 |
| `001_고도화.md` | 지금 미룬 기능과 나중에 할 일을 볼 때 |
| `002_마일스톤.md` | AI 파트를 어떤 순서로 만들었는지 볼 때 |
| `003_기획서.md` | 서비스 전체 기획을 볼 때 |
| `004_역할분배.md` | 팀원별 담당 범위를 볼 때 |
| `005_프로젝트학습문서_gg.md` | 프로젝트 전체를 공부할 때 |
| `006_문서구조_gg.md` | 문서 위치를 찾을 때 |

## 2. 브랜치별 문서

특정 작업 중에 만든 계획, 메모, 커밋 설명은 브랜치 이름 폴더에 둔다.

| 폴더 | 내용 |
| --- | --- |
| `docs-gg-0-ai-mvp-scope` | AI MVP 범위 정리와 팀원 Codex 선택지 |
| `feat-gg-12-ai-analyze` | AI 1차 제공 흐름과 PR #20 참고 내용 |
| `feat-gg-21-architecture-graph` | Architecture Draft 입력 가드레일과 이슈 #21 결정 |
| `feat-gg-42-ai-check-simulation` | Pre-Deployment Check, Design Simulation, 수정 제안 |
| `feat-gg-51-llm-provider` | LLM 설명 보강과 fallback 계획 |
| `feat-gg-62-workspace-ai-finish` | 실제 Workspace AI 패널 연결 계획 |
| `feat-gg-162-reverse-engineering` | Reverse Engineering 그릴링 결정과 구현 계획 |

## 3. private 문서

`docs/private`에는 GitHub에 올리지 않을 개인 문서를 둔다.

예:

- 발표 스크립트
- 시연 시나리오
- 커밋 코드 설명 스크립트
- 팀원에게 설명할 연결 시나리오
- 면접 준비 답안지
- 아직 공유하기 애매한 개인 메모

공유해야 하는 구현 계획이나 결정 문서는 `docs/private`에 두지 않고 `docs/gg`로 옮긴다.

## 4. private 폴더 규칙

`docs/private` 안에는 별도 `etc` 폴더를 만들지 않는다.
