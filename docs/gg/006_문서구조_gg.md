# gg 문서 구조

이 문서는 gg 담당 작업 문서를 어디서 보면 되는지 정리한 안내 문서다.

쉽게 말하면:

> 계속 보는 큰 문서는 `docs/gg` 바로 아래에 두고, 현재 작업 문서는 해당 브랜치 이름 폴더에 둔다. 예전 AI 전담 시절 문서는 `old-ai-work`에 모아둔다.

## 1. 최상위 문서

`docs/gg` 바로 아래 문서는 전체 흐름을 볼 때 읽는다.

| 문서 | 언제 읽나 |
| --- | --- |
| `000_현재작업방향_gg.md` | 새 역할 기준과 현재 담당 범위를 볼 때 |
| `003_기획서.md` | 서비스 전체 기획을 볼 때 |
| `005_프로젝트학습문서_gg.md` | 프로젝트 전체를 공부할 때 |
| `006_문서구조_gg.md` | 문서 위치를 찾을 때 |

## 2. 브랜치별 문서

특정 작업 중에 만든 계획, 메모, 커밋 설명은 브랜치 이름 폴더에 둔다.

| 폴더 | 내용 |
| --- | --- |
| `feat-gg-162-reverse-engineering` | Reverse Engineering 그릴링 결정과 구현 계획 |
| `feat-infrastructure-template` | AWS 인프라 Template 구현 패턴, 구현 설계, 배포 검증 문서 |
| `feat-gg-0-github-template` | GitHub Repository Analysis, Template Selection, AI Handoff 구현 마일스톤 |

현재 문서:

- `feat-infrastructure-template/007_AWS템플릿패턴_gg.md` — 여섯 AWS Template 패턴과 지원 가능성
- `feat-infrastructure-template/008_AWS템플릿구현설계_gg.md` — 공통 `TemplateDefinition` 구조와 배포 검증 기준
- `feat-infrastructure-template/009_AWS템플릿구현계획_gg.md` — 구현 작업과 테스트 순서
- `feat-infrastructure-template/010_AWS템플릿구현마일스톤_gg.md` — 마일스톤·커밋·검증 진행 기록
- `feat-gg-0-github-template/000_구현마일스톤_gg.md` — Repository Analysis와 Template Selection 구현·커밋 순서

## 3. old AI work 문서

예전 AI 전담 시절 브랜치 문서는 `docs/gg/old-ai-work`에 모아둔다.

이 폴더는 현재 담당 범위를 설명하는 기본 문서가 아니라, 과거 결정과 구현 흐름을 추적할 때만 읽는다.

| 폴더 | 내용 |
| --- | --- |
| `old-ai-work/top-level` | 예전 최상위 AI 범위, 고도화, 마일스톤, 역할분배 문서 |
| `old-ai-work/docs-gg-0-ai-mvp-scope` | 예전 AI MVP 범위 정리와 팀원 Codex 선택지 |
| `old-ai-work/feat-gg-12-ai-analyze` | AI 1차 제공 흐름과 PR #20 참고 내용 |
| `old-ai-work/feat-gg-21-architecture-graph` | Architecture Draft 입력 가드레일과 이슈 #21 결정 |
| `old-ai-work/feat-gg-42-ai-check-simulation` | Pre-Deployment Check, Design Simulation, 수정 제안 |
| `old-ai-work/feat-gg-51-llm-provider` | LLM 설명 보강과 fallback 계획 |
| `old-ai-work/feat-gg-62-workspace-ai-finish` | 실제 Workspace AI 패널 연결 계획 |

## 4. private 문서

`docs/private`에는 GitHub에 올리지 않을 개인 문서를 둔다.

예:

- 발표 스크립트
- 시연 시나리오
- 커밋 코드 설명 스크립트
- 팀원에게 설명할 연결 시나리오
- 면접 준비 답안지
- 아직 공유하기 애매한 개인 메모

공유해야 하는 구현 계획이나 결정 문서는 `docs/private`에 두지 않고 `docs/gg`로 옮긴다.

## 5. private 폴더 규칙

`docs/private` 안에는 별도 `etc` 폴더를 만들지 않는다.

개인 문서는 `docs/private` 바로 아래에서 번호로 찾는다.
