# ys 플랫폼 선택지

이 문서는 ys 플랫폼 파트에게 묻는 문서다.

쉽게 말하면:

> AI 분석 결과를 프로젝트 목록, 최근 작업, 알림에 어디까지 보여줄지 정하는 문서다.

## 1. 프로젝트 목록에 AI 요약을 보여줄까?

### A. MVP에서는 목록에 넣지 않기. 추천

프로젝트 목록은 가볍게 유지한다.

AI 상세 요약은 프로젝트 상세나 확인 보드에서 보여준다.

### B. 목록에 간단 요약만 넣기

예:

```text
위험도 높음
예상 비용 $20
```

좋지만 AI 결과 저장/캐싱 문제가 생긴다.

### C. 목록에 AI 상세 결과까지 넣기

목록 API가 무거워진다.

## 2. 최근 작업에 AI 이벤트를 남길까?

### A. 모든 AI 요청을 남기기

너무 많아진다.

리소스 설명 한 번 본 것도 기록될 수 있다.

### B. 중요한 AI 작업만 남기기. 추천

예:

```text
ai.architecture_draft_created
ai.pre_deployment_check_completed
ai.pre_deployment_check_failed
```

### C. AI 작업은 전혀 남기지 않기

간단하지만 사용 흐름이 덜 보인다.

## 3. 알림과 연결할까?

### A. MVP에서는 화면 warning 또는 toast만 사용. 추천

저장형 알림은 나중에 한다.

### B. high 위험만 저장형 알림

유용하지만 중복/읽음 처리 기준이 필요하다.

### C. 모든 finding을 알림으로 저장

알림이 너무 많아진다.

## 4. 로그인 전/후 소유자는 어떻게 처리할까?

### A. 로그인 사용자만 지원

인증 붙기 전 테스트가 어렵다.

### B. 익명 workspace만 지원

로그인 후 내 프로젝트 목록과 연결이 약하다.

### C. 둘 다 고려. 추천

익명 사용자는 `workspaceId` 기준.

로그인 사용자는 `userId` 기준.

## ys가 제안한 AI 요약 필드

```ts
type AiAnalysisSummary = {
  status: "not_analyzed" | "completed" | "warning" | "failed";
  highestSeverity: "low" | "medium" | "high" | null;
  findingCount: number;
  estimatedMonthlyCost: number | null;
  summary: string;
  updatedAt: string;
};
```

## gg가 맞출 것

- 프로젝트 목록에 AI 상세 요약을 꼭 넣으라고 요구하지 않는다.
- AI 요청에 AWS Access Key, Secret Key를 요구하지 않는다.
- finding에는 `id`, `category`, `severity`, `title`, `description`, `recommendation`, `resourceId`를 포함한다.
- checklist에는 `id`, `label`, `status`를 포함한다.
- 중요한 이벤트 이름은 ys와 맞춘다.

## ys에게 받고 싶은 답

```text
ys 선택 결과

1. 프로젝트 목록 AI 요약: A/B/C
2. 최근 작업 AI 이벤트: A/B/C
3. 알림 연동: A/B/C
4. 로그인 전/후 소유자 기준: A/B/C
```

## gg 추천

추천은 A / B / A / C.

즉:

- 목록은 가볍게
- 중요한 AI 이벤트만 활동 기록
- 알림은 MVP에서 가볍게
- 익명/로그인 둘 다 고려
