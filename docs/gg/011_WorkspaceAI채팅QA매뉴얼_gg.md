# Workspace AI 채팅 QA 매뉴얼

## 한 줄 안내

이 문서는 새 AI 버튼과 채팅 패널을 화면에서 직접 눌러보며 정상 동작을 확인하는 순서다.

처음에는 **열기 → 요청하기 → 미리보기 취소하기**까지만 해보면 된다. `Board에 적용`은 테스트 프로젝트에서만 누른다.

## 1. 준비

1. `http://127.0.0.1:3000`을 연다.
2. 로그인한다.
3. 대시보드에서 테스트할 프로젝트를 연다.
4. Architecture Board가 보이는 `/workspace` 화면으로 들어간다.
5. 가능하면 Resource가 여러 개 있는 테스트 프로젝트를 사용한다.

정상이라면 Board 오른쪽 아래에 검정색 네모 AI 버튼이 보인다. 반짝이 모양 icon만 있고 버튼 안에는 글자가 없다.

## 2. 닫힌 AI 버튼 확인

### 마우스로 확인

1. 검정색 AI 버튼 위에 마우스를 올린다.
2. `AI 채팅 열기`라는 작은 안내 문구가 나타나는지 본다.
3. 버튼이 가운데 아래의 확대·축소 도구를 가리지 않는지 본다.

### 키보드로 확인

1. `Tab`을 눌러 AI 버튼으로 이동한다.
2. 버튼 둘레에 focus 표시가 보이는지 확인한다.
3. `Enter` 또는 `Space`를 누른다.

정상이라면 오른쪽에서 AI 패널이 열리고, 바로 요청 입력창에 커서가 들어간다.

## 3. AI 패널 기본 화면 확인

패널이 열리면 아래 순서로 본다.

1. 위쪽에 `AI Assistant`가 보인다.
2. 그 아래에 현재 상태 이름과 짧은 설명이 보인다.
3. `설계 제안`, `오류 분석`, `에이전트 리뷰` 탭이 보인다.
4. 각 탭을 눌렀을 때 선택 상태가 분명히 바뀌는지 확인한다.
5. 세 탭의 대화 내역이 서로 섞이지 않는지 확인한다.
6. 한 탭에 입력만 남긴 뒤 다른 탭으로 이동했다가 돌아왔을 때, 입력·처리 상태·오류·승인 대기가 각각의 탭에 그대로 남는지 확인한다.
7. 대화 내용만 위아래로 움직인다.
8. 요청 입력창은 항상 패널 아래에 붙어 있다.
9. 오른쪽 위에는 닫기 `X` 버튼이 있다.

글자가 잘리거나 Board 위의 다른 버튼과 겹치면 오류다.

## 4. 닫기와 다시 열기

1. 입력창에 `닫았다 열어도 남아야 하는 입력`이라고 적는다.
2. 아직 전송하지 않는다.
3. 오른쪽 위 `X`를 눌러 패널을 닫는다.
4. AI 버튼을 다시 누른다.

정상이라면 입력한 문장과 이전 대화가 그대로 남아 있다.

이제 입력창에서 `Escape`를 눌러본다. 패널이 닫히고 focus가 AI 버튼으로 돌아오면 정상이다.

마지막으로 `오류 분석` 또는 `에이전트 리뷰`를 선택한 뒤 닫고 다시 열어본다. 일반 AI 런처로 열었을 때 마지막으로 선택한 대화가 다시 열리면 정상이다. Terraform Issue 또는 Preview 요청으로 열었을 때는 각각 `오류 분석`, `에이전트 리뷰`가 열려야 한다.

## 5. Inspector와 겹치지 않는지 확인

1. Board의 Resource 하나를 눌러 오른쪽 Inspector를 연다.
2. AI 버튼 위치를 본다.

정상이라면 AI 버튼이 Inspector 왼쪽으로 이동해 Inspector 위를 가리지 않는다.

3. AI 버튼을 누른다.

정상이라면 Inspector가 닫히고 AI 패널만 열린다.

## 6. Terraform 패널과 겹치지 않는지 확인

1. AI 패널을 닫는다.
2. Workspace 작업 도구에서 `Terraform Preview`를 누른다.
3. 열린 Terraform 패널과 AI 버튼이 겹치지 않는지 본다.

넓은 화면에서는 AI 버튼이 Terraform 패널 왼쪽으로 이동한다. 작은 화면에서는 Terraform 패널이 열린 동안 AI 버튼이 숨는다.

4. 넓은 화면에서 이동한 AI 버튼을 누른다.

정상이라면 Terraform 패널이 닫히고 AI 패널이 열린다.

저장하지 않은 Terraform 수정이 있다면 기존 이탈 확인창이 먼저 나오는 것이 정상이다.

## 7. 변경 미리보기와 승인 경계 확인

이 단계에서는 **적용하지 않고 취소**한다.

1. AI 입력창에 아래 문장을 입력한다.

```text
현재 API 앞에 ALB를 추가해줘
```

2. 전송 버튼을 누른다.
3. AI가 무엇을 추가할지 다시 묻는다면 `API 입구`를 누른다.
4. `제안 생성됨`, `적용 대기`가 보이는지 확인한다.
5. Board에 추가될 Resource가 미리보기로만 보이는지 확인한다.
6. `취소`를 누른다.

정상이라면 다음과 같이 동작한다.

- 미리보기 Resource가 사라진다.
- `제안을 취소했습니다. 실제 상태는 바뀌지 않았습니다.`라는 응답이 보인다.
- 실제 Board에는 새 Resource가 남지 않는다.

AI가 `Board에 적용`을 누르기 전에 실제 Board를 저장하거나 바꾸면 오류다.

미리보기 뒤 Board를 다른 경로로 바꾼 경우에는 기존 제안이 `오래된 제안`으로 남아야 한다. 이때 `적용`은 비활성화되고 `다시 생성`만 가능해야 한다.

## 8. 실제 적용 확인

이 단계는 버려도 되는 테스트 프로젝트에서만 진행한다.

1. 앞의 ALB 요청을 다시 보낸다.
2. `제안 생성됨`, `적용 대기`까지 진행한다.
3. `Board에 적용`을 누른다.

정상이라면 미리보기였던 Resource가 실제 Board에 남고, 대화에는 `사용자가 적용함`이라고 표시된다.

사용자가 버튼을 누르지 않았는데 적용되면 가장 중요한 오류다.

## 9. 오류 상태 확인

1. 아래처럼 Architecture와 관계없는 요청을 보낸다.

```text
오늘 날씨를 알려줘
```

정상이라면 Board를 바꾸지 않고 `요청 실패` 또는 지원 범위를 설명하는 오류가 표시된다. 오류는 빨간색만 쓰지 않고 icon, 상태 이름, 설명이 함께 보여야 한다.

## 10. 네트워크 끊김 확인

Chrome 개발자 도구를 열 수 있을 때 진행한다.

1. 개발자 도구의 `Network` 탭을 연다.
2. 연결 상태를 `Offline`으로 바꾼다.
3. AI 패널을 확인한다.

정상이라면 상태가 `연결 확인 필요`로 바뀌고 입력과 전송이 막힌다.

4. 연결 상태를 다시 `Online`으로 바꾼다.

정상이라면 다시 입력할 수 있다.

## 11. 생성 중지 확인

AI 응답이 너무 빨라 버튼을 누르기 어렵다면 개발자 도구의 Network 속도를 `Slow 3G`로 바꾼다.

1. AI 요청을 보낸다.
2. 전송 화살표가 네모 모양의 중지 버튼으로 바뀌는지 본다.
3. 중지 버튼을 누른다.

정상이라면 진행 표시가 사라지고 입력 가능한 상태로 돌아온다. 이전 대화와 이미 만든 미리보기는 지워지지 않는다.

## 12. 음성 입력 확인

음성 인식을 지원하는 브라우저에서만 마이크 버튼이 보인다.

1. 마이크 버튼을 누른다.
2. 브라우저가 권한을 물으면 테스트할 때만 허용한다.
3. 짧게 말한 뒤 중지한다.

정상이라면 들은 문장이 입력창에 들어간다. 마이크를 눌렀다는 이유만으로 요청이 자동 전송되면 안 된다.

## 13. 모바일 화면 확인

개발자 도구의 반응형 화면에서 먼저 `375 x 812`를 선택한다.

1. AI 버튼이 화면 오른쪽 아래에서 잘리지 않는지 본다.
2. AI 버튼을 누른다.
3. AI 패널이 좁은 오른쪽 칸이 아니라 전체 화면으로 열리는지 본다.
4. 위쪽 닫기 버튼과 아래쪽 입력창이 동시에 보이는지 확인한다.
5. 긴 대화를 위아래로 움직여도 입력창이 화면 아래에 남는지 본다.
6. 모바일 키보드를 열어도 입력창이 가려지지 않는지 본다.
7. `Escape` 또는 닫기 버튼으로 닫는다.

같은 검사를 `768 x 1024`에서도 반복한다.

가로 스크롤이 생기거나 버튼 글자가 잘리면 오류다.

## 14. 빠른 최종 체크

아래가 전부 맞으면 화면 QA를 통과한 것으로 본다.

- [ ] AI 버튼은 44px 정도의 검정 네모이며 tooltip이 보임
- [ ] 열면 입력창으로 focus가 이동함
- [ ] `Escape`로 닫히고 AI 버튼으로 focus가 돌아옴
- [ ] 닫았다 열어도 대화와 입력이 남음
- [ ] `설계 제안`, `오류 분석`, `에이전트 리뷰`가 각각 독립된 대화 내역을 유지함
- [ ] 세 대화의 입력·처리·오류·승인 상태도 서로 독립적임
- [ ] 일반 런처는 마지막 대화를, Terraform 요청은 해당 대화를 엶
- [ ] 오래된 설계 제안은 보이지만 적용할 수 없고 다시 생성할 수 있음
- [ ] Inspector와 Terraform 패널을 가리지 않음
- [ ] 요청 중, 완료, 미리보기, 적용 대기, 오류 상태가 서로 다르게 보임
- [ ] 사용자가 적용하기 전에는 실제 Board와 Terraform이 바뀌지 않음
- [ ] 취소하면 미리보기가 사라짐
- [ ] Offline에서는 요청할 수 없음
- [ ] 375px과 768px에서 전체 화면 패널로 동작함
- [ ] 화면에 가로 스크롤이나 잘린 글자가 없음

## 15. 오류를 발견했을 때 남길 내용

아래 네 가지만 적으면 다시 찾기 쉽다.

```text
화면 크기:
누른 순서:
기대한 결과:
실제로 나온 결과:
```

가능하면 문제가 보이는 화면도 함께 캡처한다.
