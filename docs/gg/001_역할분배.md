만약 정글 최종 프로젝트(5주, 5인) 기준이라면 이 중에서도 **반드시 구현해야 하는 기능**은 사실 10개 정도다.

1. 로그인
2. AWS 연결

1. 다이어그램 편집기
2. 리소스 패널
3. Terraform 생성
4. 코드 ↔ 다이어그램 동기화
5. Plan
6. Apply
7. 비용 추정
8. 배포 이력

여기까지가 "SketchCatch"의 핵심 가치다.

GitOps, CI/CD, 자동 삭제, 커뮤니티, AI 챗봇, 롤백 엔진까지 전부 넣으면 5주 안에 끝나기 어렵고, 심사위원 입장에서도 핵심 가치가 흐려질 가능성이 있다. 오히려 **"시각 설계 → Terraform 생성 → AWS 배포 → 비용/위험 분석"** 흐름을 완성하는 것이 우선순위가 높다.

### 본인이 해야될 것

- 만들어야 하는 기능을 정하면
- 거기에 대해 조사를 한다.
    - 어떤 기능이 필요하며
    - 어떤 기능과 의존성이 있는지
- 조사를 바탕으로 계획을 세우고..
- 공부도 하고..
- 실제로 구현 가능한지 검증하고..
- 안될 거 같으면 → 대체 방안 찾아와서 구현
- 구현까지…
- BY CODEX….
- 껍데기는 팀장님이…

과거 페이지

# 1번: 설계 → 정현

### 담당 기능

- VPC / EC2 / RDS / S3 드래그 앤 드롭
- 리소스 간 연결선 생성
- 리소스 설정 패널
- 필수 파라미터 입력 UI
- 다이어그램 저장
- 다이어그램 불러오기
- 자동 저장

### 산출물

- 사용자가 인프라를 시각적으로 설계할 수 있는 화면
- 다이어그램 JSON 데이터 구조

---

# 2번: Terraform 변환 → 시원

### 담당 기능

- 다이어그램 JSON → Terraform 코드 자동 생성
- Terraform 코드 에디터
- Terraform 코드 문법 검증
- 코드 수정 시 다이어그램 반영
- Terraform State 저장 구조 설계

### 산출물

- 실제 실행 가능한 Terraform 코드
- 코드와 다이어그램의 동기화 로직

---

# 3번: 배포 실행 → 채강

### 담당 기능

- AWS Access Key / Role Assume 연결
- AWS 연결 테스트
- Terraform Plan 실행
- Plan 결과 시각화 데이터 제공
- Terraform Apply 실행
- 실시간 배포 로그
- 배포 이력 저장
- 생성된 리소스 목록 저장

### 산출물

- 실제 AWS 배포 흐름
- 배포 성공/실패 결과
- 배포 로그 및 이력

---

# 4번: AI 분석 → 경근

### 담당 기능

- 비용 추정
- 리소스별 비용 분석
- 위험도 분석
- 보안 설정 검증
- Terraform 오류 설명
- Terraform 코드 작성 보조
- GitHub 링크 기반 초안 생성

### 산출물

- 배포 전 비용/위험 분석 결과
- 배포 실패 시 쉬운 오류 설명
- AI 기반 아키텍처/코드 보조 기능
