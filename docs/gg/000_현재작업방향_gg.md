# 현재 작업 방향

이 문서는 예전 AI 전담 문서와 구분하기 위한 현재 기준 문서다.

## 1. 최종 방향

SketchCatch는 요구사항 입력부터 Practice Architecture, IaC Preview, 안전 검증, Direct Deployment 또는 Git/CI/CD handoff, Reverse Engineering, Cost Risk, Deployment History까지 이어지는 AWS-first, Terraform-first, multi-cloud-ready IaC 운영 서비스로 완성한다.

## 2. 현재 핵심 흐름

```text
Requirement Prompt
-> Architecture Draft
-> User-Accepted Board Change
-> IaC Preview
-> Diagnostics
-> Pre-Deployment Check
-> Cost Risk
-> Terraform Plan
-> Approval
-> Direct Apply 또는 Git/CI/CD Handoff
-> Deployment History
-> Cleanup
```

Reverse Engineering은 아래 흐름으로 같은 후반 단계에 합류한다.

```text
Scan
-> Practice Architecture 복원
-> IaC Preview
-> Safety/Cost
-> Git/CI/CD 또는 Direct Deployment
```

## 3. 역할 기준

| 담당자 | 서비스 책임 | 완료 기준 |
| --- | --- | --- |
| 1번 | Requirement Input, AI Architecture Recommendation, Bedrock/Amazon Q | 텍스트/음성 요구사항이 확정 prompt가 되고, AI 제안이 승인 기반으로 보드에 반영됨 |
| 2번 | Architecture Board, Infrastructure Graph, IaC Preview | 보드 수정과 Terraform preview가 같은 설계를 바라보고, apply 가능 Resource가 안정적으로 생성됨 |
| 3번 | Deployment, Git/CI/CD Integration, Runtime Cache | Direct Deployment와 Git/CI/CD Path가 모두 plan/check/approval 기준으로 동작함 |
| 4번 | Reverse Engineering, Cost Analysis, Deployment Safety Gate | 기존 AWS Resource를 Practice Architecture로 복원하고, High risk 배포를 차단함 |

## 4. 내 담당 범위

현재 내 담당 범위는 예전처럼 AI 파트 하나로 좁히지 않는다.

주요 축은 아래와 같다.

- Provider Adapter 기반 Reverse Engineering
- Cost Risk 분석
- Deployment Safety Gate
- CheckFinding, import suggestion, risk/cost finding 통합

AI, Bedrock, Amazon Q Assistance는 이 흐름을 설명하고 보강하는 도구로 사용한다. 하지만 실제 Practice Architecture 변경, IaC handoff, Git 변경, Deployment 실행은 User-Accepted Change로만 진행한다.
