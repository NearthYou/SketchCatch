## 2. 서비스 정의

## 2.1 한 줄 소개

**SketchCatch는 자연어와 다이어그램으로 AWS 인프라를 설계하고, Terraform 코드 생성, 배포 전 검증, 비용/성능 시뮬레이션, 버전 관리를 제공하는 AI 기반 인프라 설계 플랫폼이다.**

## 2.2 서비스 설명

SketchCatch는 사용자가 AWS 콘솔에서 리소스를 하나씩 생성하거나 Terraform 코드를 처음부터 작성하지 않아도, 원하는 서비스 조건을 입력해 인프라 설계 초안을 만들 수 있도록 돕는다.

사용자는 자연어로 요구사항을 입력하거나, 다이어그램에서 직접 리소스를 배치해 인프라를 설계한다. SketchCatch는 해당 설계를 시각적 다이어그램과 Terraform 코드로 연결하고, 배포 전에 보안·비용·성능·네트워크 위험을 검토한다.

최종적으로 사용자는 설계안을 저장하고, 버전별 변경 사항을 확인하며, Terraform 코드를 export하거나 AWS 계정 연동을 통해 Plan/Apply 흐름으로 배포까지 진행할 수 있다.

## 3. 서비스 목표

| 목표 | 설명 |
| --- | --- |
| 인프라 설계 시간 단축 | 자연어 입력과 템플릿을 통해 AWS 인프라 초안을 빠르게 생성한다. |
| 구조 이해 향상 | 리소스 간 관계를 다이어그램으로 보여주어 전체 구조를 쉽게 파악하게 한다. |
| Terraform 작성 부담 완화 | 다이어그램 기반 설계를 Terraform 코드로 자동 변환한다. |
| 배포 전 위험 감소 | 보안, 비용, 성능, 네트워크 위험을 사전에 탐지한다. |

| 배포 연결성 확보 | Terraform export, Plan, Apply, Destroy, 배포 로그, Outputs 확인까지 연결한다. |
| 설계 실험 지원 | 비용 우선, 성능 우선, 보안 우선 등 여러 설계안을 비교할 수 있게 한다. |

## 4. 사용자 문제 정의

| 문제 | 사용자 상황 | 서비스가 해결할 지점 |
| --- | --- | --- |
| 인프라 구조 파악이 어렵다 | AWS 콘솔에서는 VPC, Subnet, EC2, RDS, SG 등 리소스를 개별 화면에서 관리한다. | 리소스와 연결 관계를 하나의 다이어그램으로 제공한다. |
| 설계 판단이 어렵다 | 서비스 규모, 예산, 트래픽, 보안 수준에 따라 구성 방식이 달라진다. | 자연어 요구사항을 기반으로 적합한 인프라 초안을 제안한다. |
| Terraform 작성이 어렵다 | 리소스 참조, 변수, 의존성, 출력값을 직접 작성해야 한다. | 설계 그래프를 Terraform 코드로 변환한다. |
| 배포 전 위험을 놓치기 쉽다 | SSH 전체 오픈, Public DB, CIDR 충돌, NAT 비용 등은 사전에 발견하기 어렵다. | 배포 전 검증과 AI 수정 제안을 제공한다. |
| 비용·성능 예측이 어렵다 | 설계 단계에서 병목 지점이나 비용 증가 요인을 판단하기 어렵다. | 예상 비용, 요청 흐름, 병목 가능성을 시뮬레이션한다. |
| 설계와 코드가 분리된다 | 다이어그램은 문서에 있고 Terraform 코드는 별도 파일로 관리된다. | 다이어그램과 Terraform 코드를 같은 설계 단위로 연결한다. |
| 변경 이력 추적이 어렵다 | 어떤 리소스가 언제, 왜 변경되었는지 파악하기 어렵다. | 버전 관리, diff, rollback preview를 제공한다. |

인프라 설계의 핵심 어려움은 리소스 관계 파악, 설계 경험 요구, 보안·비용 검증의 어려움으로 정리된다.

## 9.2 AI 기반 설계 생성 플로우

```
자연어 입력
→ 요구사항 분석
→ 설계 후보 생성
→ 추천 구조 미리보기
→ 사용자가 적용
→ 다이어그램 생성
→ Terraform 코드 생성
→ 배포 전 검증 실행
```

### 사용자 입력 예시

```
서울 리전에 운영용 3-tier 웹 서비스를 만들어줘.
CloudFront와 S3, VPC 안의 public/private subnet, private EC2 백엔드,
RDS PostgreSQL을 사용하고 배포 전 비용과 보안 위험을 검토해줘.
```

### AI가 해석해야 하는 항목

| 항목 | 설명 |
| --- | --- |
| 서비스 목적 | 웹 서비스, API 서버, 정적 웹사이트 등 |
| 리전 | AWS 리전 |
| 리소스 조건 | CloudFront, S3, EC2, RDS 등 |
| 네트워크 조건 | Public/Private Subnet, VPC 구성 |
| 보안 조건 | DB 외부 노출 금지, SSH 제한 등 |
| 비용 조건 | 월 예산, 고비용 리소스 최소화 |
| 성능 조건 | 트래픽, 병목 가능성, 확장성 |
| 배포 조건 | Terraform 생성, 검증, Plan/Apply |

| 결과 영역 | 표시 내용 |
| --- | --- |
| 설계 요약 | 생성된 아키텍처 목적, 주요 리소스, 적용된 우선순위 |
| 리소스 목록 | VPC, Subnet, ALB, EC2/ECS, RDS, S3, SG 등 |

## 9.3 다이어그램 직접 설계 플로우

```
리소스 선택
→ 캔버스에 배치
→ 리소스 연결
→ 속성 입력
→ 검증 실행
→ Terraform 코드 확인
```

### 주요 사용자 행동

| 행동 | 시스템 반응 |
| --- | --- |
| 리소스 추가 | 선택한 리소스가 캔버스에 생성된다. |

| 리소스 연결 | 연결 가능한 리소스만 연결된다. |
| 리소스 삭제 | 연결된 리소스 영향 범위를 안내한다. |
| 속성 수정 | 비용, 보안, 코드 변경 영향이 표시된다. |

| --- | --- |

## 11.11 AWS 계정 연동

### 기능 목적

사용자 AWS 계정에 안전하게 접근해 Plan/Apply/Destroy를 실행한다.

### 사용자 플로우

```
AWS 계정 연결
→ IAM Role 생성 안내
→ Role ARN 입력
→ 권한 확인
→ 연결 완료
→ 프로젝트 배포 대상 계정으로 선택
```

### 연동 화면 구성

| 영역 | 내용 |
| --- | --- |
| 계정 목록 | 연결된 AWS 계정 |
| 권한 상태 | 정상, 권한 부족, 만료 |
| 리전 설정 | 기본 배포 리전 |
| IAM Role 정보 | Role ARN, 마지막 확인 시각 |
| 보안 안내 | 필요한 권한과 위험 안내 |
| 연결 해제 | 계정 연동 해제 |

### 권한 정책

| 정책 | 설명 |
| --- | --- |
| Access Key 직접 저장 금지 | 장기 Access Key 입력 방식은 제공하지 않는다. |
| IAM Role 기반 접근 | Role ARN 기반 연결을 기본으로 한다. |
| 최소 권한 안내 | 필요한 권한 범위를 명확히 표시한다. |
| 배포 전 재확인 | Apply/Destroy 실행 전 권한과 대상 계정을 다시 표시한다. |

# 17. 정책성 문구

## 17.1 서비스 소개 문구

**SketchCatch는 자연어와 다이어그램으로 AWS 인프라를 설계하고, Terraform 코드 생성, 배포 전 검증, 비용·성능 시뮬레이션, 버전 관리까지 연결하는 AI 기반 인프라 설계 플랫폼입니다.**

## 17.2 상세 소개 문구

SketchCatch는 클라우드 인프라 설계 과정에서 발생하는 구조 파악, 코드 작성, 비용 예측, 보안 검증, 변경 관리의 부담을 줄이는 서비스입니다. 사용자는 자연어로 요구사항을 입력하거나 다이어그램에서 리소스를 직접 배치해 인프라를 설계할 수 있습니다. SketchCatch는 설계 결과를 Terraform 코드로 변환하고, 배포 전에 위험 요소를 검토하며, AWS 계정 연동을 통해 Plan/Apply 흐름까지 지원합니다.

## 17.3 차별화 문구

**SketchCatch는 단순 다이어그램 도구가 아니라, 인프라 설계안을 코드·검증·시뮬레이션·배포·버전 관리까지 연결하는 서비스입니다.**

| rollback | 설계 rollback과 실제 배포 rollback을 어떻게 구분할지 |
| Drift | 실제 리소스 변경을 설계에 반영하는 UX를 어떻게 설계할지 |
| 멀티클라우드 | AWS 안정화 후 어느 클라우드부터 확장할지 |
| Kubernetes | EKS 중심으로 갈지, 일반 Kubernetes 리소스 중심으로 갈지 |

# 19. 최종 정리

SketchCatch의 서비스 범위는 다음 흐름으로 정리한다.

```
자연어 요구사항 입력
→ AI 인프라 설계안 생성
→ 다이어그램 기반 수정
→ Terraform 코드 생성
→ 배포 전 검증
→ AI 수정안 적용
→ 비용·성능 시뮬레이션
→ 버전 저장
→ GitHub 연동
→ AWS Plan/Apply
→ 배포 로그/Outputs 확인
→ Drift/모니터링/알림
```

비지원 범위는 **실시간 공동 편집**과 **CloudFormation 동시 지원**으로 한정한다. 나머지 기능은 제외하지 않고, 제공 단계에 따라 순차적으로 확장한다.
