# AWS 가져오기 권한 복구와 Board 자동 정리 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` to implement this plan task-by-task. Every behavior change follows `superpowers:test-driven-development` and each task ends in a focused commit.

**Goal:** 기존 AWS 연결과 배포 권한을 그대로 둔 채 Reverse Engineering 조회 권한을 안전하게 준비·확인·정리하고, 가져온 원본을 먼저 보여준 뒤 사용자가 요청한 경우에만 최대 3개의 시각 전용 Board 정리안을 비교·적용할 수 있게 한다.

**Architecture:** AWS 권한 복구는 connection별 Manager Stack과 Policy Stack, 별도 DB 상태, 실제 read probe를 Settings가 소유한다. Board 자동 정리는 원본 의미 위에 허용된 geometry만 다시 조립하는 깊은 모듈로 만들고 Reverse Engineering은 그 공개 인터페이스만 사용한다. 공개 API와 UI에는 AWS 원문 오류, Policy JSON, Compiler 점수와 내부 ID를 전달하지 않는다.

**Tech Stack:** TypeScript, Fastify, Drizzle/PostgreSQL, AWS SDK v3, Next.js/React, TanStack Query, Node test runner, React server rendering tests.

## Global Constraints

- 기존 `AwsConnection` ID, account, region, Role ARN, 원래 CloudFormation Stack, 배포 Policy, 배포용 `verified` 상태를 바꾸지 않는다.
- Manager Stack 생성·갱신과 두 Stack 삭제는 AWS Console에서 사용자가 직접 승인한다.
- Policy Stack 생성·갱신은 Settings에서 10분 수명의 일회성 승인을 받은 뒤에만 실행한다.
- Reverse Engineering은 AWS 권한을 변경하지 않고 성공한 Resource와 관계를 항상 보존한다.
- 자동 정리는 Resource, 관계, 설정, 실제 containment, Terraform 의미를 변경하지 않는다.
- 잠긴 요소와 잠긴 자동 표시 프레임의 geometry를 변경하지 않는다.
- 후보 선택과 미리보기는 Board를 저장하지 않으며 `이 정리 사용`만 한 번 저장한다.
- 공개 응답과 UI에 ARN, AWS action, Request ID, SDK message, Policy JSON, Stack logical ID, Compiler 점수·version·candidate ID·Template ID를 보여주지 않는다.
- 사용자 문구는 짧고 쉬운 한국어로 작성한다.
- 새로 만들거나 수정하는 gg 함수와 메서드에는 역할이나 안전 경계를 설명하는 짧은 주석을 붙인다.
- 실제 AWS Resource 생성·수정·삭제, Terraform Apply·Destroy, Deployment 실행은 검증 과정에서 수행하지 않는다.
- CloudFormation API 제약상 IAM capability와 `ResourceTypes`는 함께 전달하지 않는다. 대신 exact Stack ARN, immutable `TemplateURL`, exact service Role, ownership tag, 최소 권한 service Role, 생성 후 Template hash 검증을 함께 적용한다.
- 원격 `dev`의 최신 migration `0053`을 확인했으므로 이번 작업은 `0054`를 사용한다. 파일 생성 직전에 원격을 다시 확인해 더 새 번호가 있으면 다음 번호로 조정한다.
