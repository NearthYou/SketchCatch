AI가 모든 걸 알아서 하게 둔 게 아니라, 안전한 보조 기능으로 제한했습니다.

팀원이 몰라도 되는 핵심은 이것입니다. AI는 결정을 대신하는 엔진이 아니라, deterministic data를 설명하고 위험을 초보자 언어로 바꿔주는 계층입니다.

초안 제안, 비용/보안/설정 finding 설명, Terraform 오류 번역, IaC Preview 요약.

gg AI API는 5개입니다.

버튼을 눌러 endpoint별 역할과 팀 연결 지점을 확인하세요.

/api/ai/architecture-draft

자연어 prompt를 받아 ArchitectureJson 기준의 Architecture Draft를 반환합니다.

POST /api/ai/architecture-draft
body: { "prompt": "DB 포함 백엔드 API 서버" }

5 / 10

팀원 입장에서 봐야 할 지점은 각자 다릅니다.

각 이니셜을 눌러 “내 파트가 무엇을 맞추면 되는지”만 빠르게 보세요.

jh: Architecture Board

AI가 만든 ArchitectureJson을 보드에서 열 수 있으면 됩니다. Finding은 resourceId로 보드 노드와 연결됩니다.

ArchitectureJson.nodes[].id === CheckFinding.resourceId

6 / 10

공통 계약은 `packages/types`에서 맞췄습니다.

팀원들이 코드에서 가장 먼저 확인해야 할 타입은 아래 4개입니다.

AiArchitectureDraftResult

Architecture Board가 열 수 있는 ArchitectureJson과 AI 설명 metadata를 함께 반환합니다.

type AiArchitectureDraftResult = {
  architectureJson: ArchitectureJson;
  title: string;
  metadata: AiResultMetadata;
};
