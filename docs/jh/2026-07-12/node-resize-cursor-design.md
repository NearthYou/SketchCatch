# 노드 크기 조절 커서 구분 설계

## 문제

영역 및 리소스 노드의 크기 조절 핸들에는 방향별 커서가 선언되어 있었지만, 전역 `button:not(:disabled)` 규칙이 이를 덮어 일반 포인터로 표시했다. 또한 크기 조절 핸들이 선택된 노드에만 렌더링되어 선택 전에는 영역 테두리의 이동 커서만 보였다.

## 결정

- 이동 가능한 영역 테두리는 기존 `grab` 커서를 유지한다.
- 실제 크기 조절 지점만 방향에 맞는 `ns-resize`, `ew-resize`, `nwse-resize`, `nesw-resize` 커서를 사용한다.
- 선택 전에는 네 모서리만 호버 시 노출해 이동과 크기 조절을 구분한다.
- 선택 후에는 기존처럼 네 면과 네 모서리를 모두 노출한다.
- 선택되지 않은 숨은 핸들은 탭 순서에서 제외한다.
- CSS 선택자 우선순위를 전역 버튼 커서보다 높여 계산된 스타일에서도 방향별 커서가 유지되게 한다.

## 검증 기준

- 모서리 호버 시 대각선 크기 조절 커서가 표시된다.
- 선택된 노드의 상·하단은 세로, 좌·우측은 가로 크기 조절 커서를 표시한다.
- 영역의 이동 지점은 `grab` 커서를 유지한다.
- 잠금, Preview, 연결 중 상태에는 크기 조절 핸들이 활성화되지 않는다.
