# 캔버스 왼쪽 세로 툴바 설계

## 목표

Architecture Board 하단 중앙의 가로 툴바를 캔버스 왼쪽 세로 중앙에 배치해, 하단 작업 영역을 가리지 않으면서 도구를 한 방향으로 빠르게 탐색할 수 있게 한다.

## 확정 배치

- 툴바는 `workspace` 내부 왼쪽에서 16px 떨어진 위치에 고정한다.
- 세로 위치는 캔버스 중앙을 기준으로 맞춘다.
- 선택된 미리보기처럼 하나의 바깥 표면 안에 세 개의 세로 그룹을 배치한다.
- 그룹 순서는 위에서 아래로 `편집 도구`, `History`, `Viewport`를 유지한다.
- 그룹 내부 아이콘 순서, `aria-label`, 클릭 동작, disabled 상태, 선택 상태, 단축키 동작은 변경하지 않는다.

## 시각 규칙

- 기존 흰색 표면, 옅은 테두리, 8px 모서리, 절제된 그림자를 그대로 사용한다.
- 바깥 툴바와 각 그룹의 가로 폭은 콘텐츠에 맞추되 모든 아이콘이 한 열로 정렬되게 한다.
- 그룹 사이는 기존 6px 간격으로 구분하고, 그룹 내부는 기존 2px 간격과 3px 패딩을 유지한다.
- 버튼 크기와 아이콘 크기는 바꾸지 않아 기존 클릭 영역과 시각 밀도를 유지한다.
- 선택, hover, focus-visible, active, disabled 표현은 기존 `iconButton` 계열 스타일을 그대로 사용한다.

## 반응형 동작

- 데스크톱과 모바일 모두 왼쪽 세로 배치를 유지한다.
- 작은 화면에서는 기존 좌우 10px 안전 여백 규칙을 왼쪽 여백에 적용한다.
- 툴바는 캔버스 위에 떠 있지만 Board 노드와 연결선 자체의 좌표나 저장된 viewport를 변경하지 않는다.

## 검토한 방식

1. 기존 마크업을 유지하고 CSS flex 방향과 위치만 변경하는 방식: 동작 회귀 위험이 가장 낮아 채택한다.
2. 세 그룹을 각각 독립 플로팅 카드로 분리하는 방식: 시각적으로 가볍지만 위치·포커스 관리가 분산되어 제외한다.
3. 왼쪽 패널 내부에 도킹하는 방식: 패널 접힘 상태와 도구 접근성이 결합되므로 제외한다.

## 접근성 및 검증

- 툴바는 기존 `header`, 그룹 레이블, 버튼 접근성 이름을 유지한다.
- 키보드 탭 순서는 화면의 위→아래 아이콘 순서와 일치해야 한다.
- 선택/이동 모드, 영역 자동 확장, Undo/Redo, Zoom, Fit view 동작이 이전과 동일해야 한다.
- 회귀 테스트는 세로 flex 방향, 왼쪽 중앙 고정, 모바일 왼쪽 여백, 기존 세 그룹과 버튼 계약을 확인한다.
- 실제 브라우저에서 데스크톱과 모바일 크기로 겹침, 포커스 표시, disabled 상태를 확인한다.
