# 자동 정리는 최대 3개의 안전한 후보를 갤러리로 제공

Board Auto Arrange는 Resource·관계·설정·containment를 보존한 서로 다른 시각 후보를 최대 3개까지 제공한다. 원본은 후보 점수 경쟁에 넣지 않고 비교 기준으로만 사용한다. geometry와 edge route가 사실상 같은 결과는 하나로 합치며, 안전한 후보가 1개나 2개뿐이면 있는 만큼만 보여준다.

데스크톱은 전체 보드 thumbnail 갤러리에서 후보를 고르고 같은 화면 아래에서 원본과 선택한 정리안을 나란히 비교한다. 모바일은 가로 thumbnail 갤러리에서 후보를 고른 뒤 같은 viewport와 zoom을 유지하는 `원본`/`정리안` 전환으로 한 화면씩 본다. 각 후보는 대표 변경 한 줄과 상세 설명을 가지며 내부 점수·후보 ID·Compiler version은 노출하지 않는다.

후보 선택과 전환은 Board를 변경하지 않는다. 사용자가 선택한 후보에서 `이 정리 사용`을 눌렀을 때만 저장하고, 안전한 시각 diff가 하나도 없을 때만 결과 없음으로 처리한다. 이 결정은 ADR 0022의 품질 비차단 원칙을 여러 후보 선택 UI로 구체화한다.
