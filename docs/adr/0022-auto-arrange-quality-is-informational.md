# 자동 정리 품질 측정값은 결과를 차단하지 않음

Board Auto Arrange는 Resource·관계·설정·containment를 보존한 시각 변경이 있으면 layout finding이나 내부 품질 점수가 개선되지 않아도 사용자에게 미리보기를 제공하고 적용을 허용한다. 서로 다른 안전한 후보가 여러 개면 원본을 점수 경쟁에 넣어 하나만 고르지 않고 여러 정리안을 제공한다. 측정값은 변경 후보의 순서를 정하고 좋아진 점과 나빠진 점을 쉬운 문장으로 설명하는 evidence일 뿐 성공 gate가 아니며, 실제 시각 diff가 없을 때만 적용할 결과가 없는 것으로 처리한다.

겹침·경계 이탈·연결선 문제 측정은 유용하지만 화면 크기와 사용 목적에 따른 모든 시각적 선호를 대신하지 못한다. SketchCatch는 안전한 변경 범위를 코드로 제한하고, 그 안에서 원본과 여러 정리안을 비교해 선택하는 최종 판단은 사용자에게 둔다. 후보 전환은 Board를 바꾸지 않고 선택한 정리안의 명시적 적용만 저장한다. 내부 점수·후보 ID·Compiler version은 사용자 화면에 노출하지 않는다.
