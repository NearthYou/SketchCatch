# Board 자동 정리는 시각적 배치만 변경

Board의 `자동 정리`는 Resource, 관계, 설정, containment 의미를 변경하지 않는다. 사용자가 요청했을 때 위치, 크기, 표현용 영역 배치, 연결선 경로만 정리한 미리보기를 만들고, 사용자가 명시적으로 선택한 뒤에만 Board에 적용한다.

Architecture Board Compiler 자체는 ADR 0011에 따라 의미와 시각 표현을 함께 바꾸는 제안을 만들 수 있다. 그러나 그 넓은 변경 권한을 `자동 정리`라는 이름으로 제공하면 사용자가 단순 배치 기능으로 이해한 채 구조 변경까지 승인할 수 있다. 따라서 Board Auto Arrange 호출은 시각적 변경만 허용하고, Resource·관계·설정 변경은 별도의 구조 개선 흐름이 생기기 전까지 사용자 기능으로 제공하지 않는다.

이 결정은 ADR 0011의 Compiler 계산 능력을 폐기하지 않고, Board Auto Arrange에서 허용하는 변경 범위를 제한한다.

Reverse Engineering 결과에는 원래 Board 좌표가 없으므로, 가져온 `원본`도 Resource·관계·설정을 보존한 채 화면에 놓기 위한 결정론적 기본 위치는 계산한다. 이 최초 배치는 Board Auto Arrange 실행이나 의미 변경 제안으로 취급하지 않는다.

미리보기의 선택은 `원본 유지`와 `이 정리 사용`으로 제한한다. `원본 유지`는 아무 변경 없이 미리보기를 닫고, `이 정리 사용`은 정리 결과를 적용하는 단일 명시적 승인이다. 같은 결정을 묻는 추가 확인창은 두지 않는다.
