## 추가 규칙

- 새 ADR은 다음 번호를 사용한다.
- 이미 결정된 내용을 바꿀 때는 기존 ADR을 조용히 덮어쓰지 말고 새 ADR에서 변경 이유와 이전 결정과의 관계를 남긴다.
- ADR 내용이 제품 범위, 데이터 모델, 실행 경계를 바꾸면 `docs/product.md`, `docs/data-models.md`, `docs/architecture.md`도 함께 갱신한다.
