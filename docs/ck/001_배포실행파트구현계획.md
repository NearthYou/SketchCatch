## 3. MVP 완료 기준

아래 조건이 만족되면 CK 배포 실행 파트의 MVP 구현은 완료로 본다.

- 특정 `projectId`, `architectureId`, `terraformArtifactId` 기준으로 mock 배포 기록을 만들 수 있다.
- 배포 기록에서 plan summary, blocked 정보, 승인 상태를 조회할 수 있다.
- blocked 상태에서는 승인 또는 실행 흐름이 차단된다.
- mock 로그를 저장하고 조회할 수 있다.
- 프로젝트별 배포 이력을 조회할 수 있다.
- 프론트엔드에서 배포 이력, 승인 상태, 로그를 확인할 수 있다.
- 실제 AWS 리소스 생성, 수정, 삭제 로직은 포함되어 있지 않다.
