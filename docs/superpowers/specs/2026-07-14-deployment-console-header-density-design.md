# 배포 콘솔 상단 탐색 및 Stepper 크기 조정 설계

## 목적

배포 콘솔 모달에서 중복된 제목 정보를 제거하고, 사용자가 `배포`와 `CI/CD` 실행 경로를 더 빠르게 구분하도록 상단 탐색의 위치와 대비를 개선한다. Direct Deployment의 단계 번호도 한눈에 읽히도록 확대한다.

## 변경 범위

- 확장 모달의 `IaC Operations`, `배포 콘솔`, 프로젝트명으로 구성된 제목 행을 DOM에서 제거한다.
- `배포`와 `CI/CD` 선택 영역을 모달 최상단 중앙에 배치한다.
- 기존 세그먼트 버튼의 활성·비활성 색상과 상태 전달 방식은 유지하되, 글자 크기와 최소 클릭 영역을 키운다.
- 닫기 버튼은 우측 상단에 유지하고 중앙 탐색과 겹치지 않도록 상단 행에 충분한 좌우 여백을 둔다.
- Stepper 원은 데스크톱 40px, 모바일 36px로 확대하고 번호와 체크 아이콘도 함께 키운다.
- 커진 원의 중심에 맞춰 연결선 위치와 길이를 다시 맞춘다.

## 유지 사항

- `배포`와 `CI/CD`의 선택 상태, `localStorage`, 콜백, 접근성 속성은 변경하지 않는다.
- 각 배포 단계의 콘텐츠, 버튼 위치, API 호출, 승인 및 실행 기능은 변경하지 않는다.
- 기존 Workspace 색상 토큰, 글꼴, 테두리, 모서리 스타일을 재사용한다.
- 모바일에서도 두 경로 선택 버튼과 세 단계가 가로 스크롤 없이 보여야 한다.

## 검증

- 소스 회귀 테스트로 제목 행 제거, 중앙 탐색, 커진 Stepper 치수와 연결선 정렬을 확인한다.
- Web lint, typecheck, build와 harness 검사를 실행한다.
- 데스크톱과 390px 모바일 폭에서 상단 탐색, 닫기 버튼, Stepper 겹침과 잘림을 확인한다.
