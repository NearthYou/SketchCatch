# Terraform AI 액션 버튼 컴팩트 정렬 설계

## 목적

Terraform Preview의 `에이전트 리뷰`와 Terraform 진단 카드의 `오류 분석` 버튼을 주변 컨트롤 및 텍스트 계층에 맞는 컴팩트한 보조 액션으로 정렬한다. 기존 AI 대화 연결, 접근성 속성, 클릭 동작은 유지한다.

## 검토한 방식

1. **컴팩트한 검정 액션 유지 — 채택**
   - 버튼 높이 `32px`, 글자 `12px`, AI 마크 `20px`로 축소한다.
   - 검정 채움과 `8px` radius를 유지해 AI 진입점이라는 공통 성격을 보존한다.
   - hover는 색상만 바꾸고 위치 이동을 제거한다.
2. **흰색 테두리 버튼으로 전환 — 제외**
   - 주변 보조 버튼과 잘 섞이지만 AI 액션의 공통 식별력이 약해진다.
3. **아이콘 전용 버튼으로 전환 — 제외**
   - 폭은 줄지만 `에이전트 리뷰`와 `오류 분석`의 서로 다른 목적을 즉시 구분하기 어렵다.

## 시각 규칙

- 두 버튼은 동일한 높이, 글자 크기, 마크 크기, 간격을 사용한다.
- 버튼 높이: `32px`
- AI 마크: `20px × 20px`
- 글자 크기: `12px`
- 버튼 내부 간격: `6px`
- 좌우 padding: AI 마크 쪽 `5px`, 텍스트 쪽 `10px`
- hover에서 `transform`을 사용하지 않는다.
- focus ring과 reduced-motion 대응은 유지한다.
- Terraform 툴바에서는 파일 선택 버튼의 수직 중앙에 정렬한다.
- 진단 카드에서는 메타데이터와 한 행에 놓이되, 좁은 폭에서는 기존처럼 자연스럽게 줄바꿈한다.

## 구현 범위

- `TerraformAgentReviewButton.module.css`
- `TerraformIssueAnalysisButton.module.css`
- 필요하면 진단 footer의 교차축 정렬만 최소 조정한다.
- JSX, 이벤트, API, AI Chat conversation scope는 변경하지 않는다.

## 검증

- 스타일 테스트가 두 버튼의 `32px` 높이, `20px` 마크, `12px` 글자, hover transform 부재를 검증한다.
- 테스트가 기존 상태에서 먼저 실패하는 것을 확인한 뒤 CSS를 수정한다.
- 관련 테스트, Web lint, Web typecheck를 실행한다.
- 실제 Workspace 우측 패널에서 툴바와 진단 카드의 정렬 및 가로 넘침을 확인한다.

## 완료 조건

- 두 AI 액션이 주변 UI보다 과도하게 크지 않다.
- hover 시 버튼 위치가 움직이지 않는다.
- 텍스트와 AI 마크가 두 버튼에서 동일한 기준으로 정렬된다.
- 기존 클릭, 키보드 focus, AI 대화 진입 동작이 유지된다.
