# 배포 이력 재설계

## 목표

중복된 `최근 실행 결과` 카드를 제거하고, 배포 이력을 큰 테이블과 선택 상세 패널로 재구성해 상태와 변경 내용을 빠르게 읽게 한다.

## 승인된 화면

- 헤더는 `배포 이력`, 간단한 설명, 전체 건수만 표시한다.
- 왼쪽 테이블은 상태, 실행 시각, 실제 변경, 실행 범위를 최소 16px로 표시한다.
- 선택한 배포는 오른쪽 상세 패널에서 상태, 결과 문장, 범위, 변경, 앱 릴리스, 버전 ID를 표시한다.
- 값이 0인 변경은 요약 문장에서 생략한다.
- Commit, digest, Build artifact는 접힌 기술 정보에 둔다.
- 리소스·Output과 로그는 기존 상세 데이터를 재사용한다.
- 성공한 Destroy의 삭제 수는 실패색으로 표시하지 않는다.

## 제거 범위

- `최근 실행 결과` 카드와 전용 레이아웃 영역
- 버전 선택 전용 드롭다운 카드
- `성공한 배포만 표시`, `resources`, `APPLICATION RELEASE` 같은 중복·영문 중심 표현

## 유지 범위

- Deployment 상태 계산과 자동 갱신
- 기존 Terraform Plan, Apply, Destroy, Rollback 동작
- 배포 이력 선택 상태와 리소스·Output·로그 조회
- 실패 메시지와 실행 단계 경고

## 반응형·접근성

- 데스크톱은 테이블과 상세 패널을 나란히 배치한다.
- 좁은 화면은 상세 패널을 테이블 아래로 내리고 테이블은 수평 스크롤을 허용한다.
- 선택 버튼은 44px 이상의 클릭 영역과 명확한 `aria-current`를 제공한다.
