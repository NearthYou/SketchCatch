# 배포 콘솔 상단 Stepper 설계

## 목적

이 변경은 사용자가 Direct Deployment의 현재 위치와 완료된 단계를 모달 상단에서 한눈에 확인하게 한다. 배포 상태 계산, 단계별 콘텐츠, 승인·배포 버튼, API 호출은 바꾸지 않고 기존 왼쪽 단계 내비게이션만 상단 가로 Stepper로 옮긴다.

## 범위

- `DirectDeploymentScreen`의 `검증`, `승인`, `배포` 3단계를 가로 Stepper로 표시한다.
- Stepper는 배포 콘솔 제목과 `배포`/`CI/CD` 경로 탭 아래, Direct Deployment 본문 위에 배치한다.
- 중앙 단계 콘텐츠와 우측 배포 컨텍스트 패널의 내용, 순서, 버튼 위치와 동작을 유지한다.
- 기존 `getDirectDeploymentFlow`가 반환하는 단계 상태와 `selectedDirectStepId`를 그대로 사용한다.
- CI/CD 화면, 배포 기록, 확인 모달, API·DTO·저장 구조는 변경하지 않는다.
- 이 UI는 실제 배포를 새로 실행하지 않는다. 기존 버튼을 통해 이미 연결된 승인·배포 동작으로 이동하는 단계 안내만 바꾼다.

## 화면 구조

Direct Deployment 영역은 다음 두 행으로 구성한다.

1. 상단 Stepper: 세 단계와 연결선을 한 행에 표시한다.
2. 작업 영역: 기존 단계 콘텐츠와 우측 배포 컨텍스트를 두 열로 유지한다.

기존 왼쪽 단계 내비게이션 열은 제거한다. Stepper 외의 제목, 경로 탭, 단계 본문, 요약 정보, 실행 버튼은 기존 디자인과 배치를 유지한다.

## Stepper 표현

각 단계는 원형 상태 표시, 단계명, 짧은 상태 문구로 구성한다.

- 완료: 원 안에 체크 아이콘을 표시하고 성공 색상을 사용한다.
- 현재: 원 안에 단계 번호를 표시하고 `--workspace-accent` 배경과 흰색 글자를 사용한다.
- 진행 중: 현재 단계와 같은 주 강조색을 사용하되 기존 상태 문구로 진행 중임을 설명한다.
- 미진행: 흰색 또는 muted surface 배경, 회색 테두리와 회색 번호를 사용한다.
- 경고: 기존 경고 색상을 유지한다.
- 차단·오류: 기존 오류 색상을 유지한다.

완료와 현재 상태가 시각적으로 겹치지 않도록 완료 단계는 항상 체크 아이콘으로 표시한다. 색상만으로 상태를 구분하지 않고 번호·체크 아이콘·상태 문구를 함께 제공한다.

## 연결선

인접한 단계 사이에 수평 연결선을 둔다.

- 앞 단계가 완료되었으면 다음 단계 방향의 연결선을 성공 색상으로 활성화한다.
- 현재 단계에 도달한 구간은 활성 상태로 표시한다.
- 아직 도달하지 않은 구간은 `--workspace-line-strong` 계열의 비활성 색상을 사용한다.
- 연결선은 원형 상태 표시의 중심을 정확히 잇고, 단계 텍스트 뒤를 지나가지 않는다.

## 상호작용

- 기존처럼 완료된 단계와 현재 단계는 클릭해 내용을 다시 볼 수 있다.
- `idle` 단계는 기존처럼 비활성화해 앞 단계가 끝나기 전에 이동할 수 없게 한다.
- 현재 실행 단계는 `aria-current="step"`을 유지한다.
- 키보드 포커스는 기존 디자인 토큰을 사용한 명확한 focus ring으로 표시한다.
- Stepper 클릭은 표시할 단계만 바꾸며 승인, Apply, Destroy를 자동 실행하지 않는다.

## 반응형 동작

- 데스크톱에서는 세 단계를 같은 폭으로 배치하고 연결선을 유지한다.
- 좁은 화면에서는 단계명과 원형 표시를 유지하되 상태 문구를 숨길 수 있다.
- 작은 모바일 화면에서도 Stepper가 잘리지 않도록 내부 여백과 원 크기를 줄인다.
- 단계 순서는 항상 `검증 → 승인 → 배포`를 유지한다.
- 본문과 우측 컨텍스트의 기존 반응형 전환 규칙은 유지한다.

## 디자인 시스템 적용

- 배경, 테두리, 텍스트, 강조색은 `workspace.module.css`의 기존 `--workspace-*` 토큰을 사용한다.
- Pretendard 기반 기존 글꼴, 8px~12px radius 체계, 기존 여백 리듬을 유지한다.
- 새로운 gradient, shadow, 별도 브랜드 색상, 장식 애니메이션은 추가하지 않는다.
- 상태 변화가 필요한 경우에만 150~200ms 범위의 색상 전환을 사용하고 `prefers-reduced-motion`에서는 제거한다.

## 구현 경계

- `DirectDeploymentScreen.tsx`: 완료 단계의 체크 아이콘과 연결선 상태를 표현하는 최소 마크업만 변경한다.
- `workspace.module.css`: 세로 내비게이션을 상단 Stepper와 2열 작업 영역으로 바꾸고 상태별 스타일과 반응형 규칙을 추가한다.
- `workspace-right-panel-layout.test.ts`: 상단 배치, 원형 상태, 연결선, 완료 체크, 모바일 규칙을 검증하도록 갱신한다.
- `deployment-console-state.ts`와 API 코드는 현재 상태 계약으로 충분하므로 변경하지 않는다.

## 검증

- 구조 테스트에서 Stepper가 작업 영역 위 전체 너비를 차지하는지 확인한다.
- 상태별 테스트에서 완료 체크, 현재 단계 강조, 미진행 비활성, 경고·오류 표현을 확인한다.
- 연결선이 완료 구간과 미진행 구간에 서로 다른 토큰을 사용하는지 확인한다.
- 키보드 탐색과 `aria-current`, disabled 상태가 유지되는지 확인한다.
- 1100px 이하와 720px 이하에서 콘텐츠가 겹치거나 잘리지 않는지 확인한다.
- `pnpm harness:check`, `pnpm lint`, `pnpm typecheck`, `pnpm build`를 실행한다.
