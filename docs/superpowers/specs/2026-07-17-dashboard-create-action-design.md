# Dashboard 새 프로젝트 행동 버튼 디자인

## 목표

Dashboard의 `작업 현황`과 `내 프로젝트` 페이지 헤더에 있는 `새 프로젝트` 행동을 유지하면서, 순검정 채움 때문에 주변의 흰 카드·얇은 구분선보다 과하게 튀는 문제를 줄인다.

## 결정

- 기존 `dashboardPrimaryAction` 전역 규칙은 유지한다.
- 두 페이지 헤더 링크에 `dashboardCreateAction` modifier를 추가한다.
- modifier는 흰 배경, 중립색 테두리, 약한 그림자와 회색 hover를 사용한다.
- hover에서 위로 이동하지 않게 해 고정된 페이지 헤더의 일부처럼 보이게 한다.
- 링크 경로, 문구, 아이콘, 모바일 표시 방식은 바꾸지 않는다.

## 검증

- 두 헤더 링크에 modifier가 모두 적용되는지 확인한다.
- 전용 스타일이 검은 배경을 제거하고 hover 이동을 제거하는지 회귀 테스트로 확인한다.
- Web typecheck와 lint를 실행한다.
