# 워크스페이스 리소스 시각 동작 설계

## 목표

workspace 페이지의 다이어그램 리소스 표시와 영역 포함 동작을 정리한다. 일반 아이콘 리소스는 더 작고 단정하게 생성하고, 영역 안으로 들어온 아이콘이 중첩된 모든 부모 영역에 충분한 여백을 갖도록 자동 확장한다. Terraform 식별자와 보드 표시명은 분리한다.

## 범위

- `apps/web/app/workspace`에서 사용하는 다이어그램 편집기만 변경한다.
- 일반 아이콘 리소스의 생성 크기, 표시명, 아이콘 프레임, 선택 크기 조절점을 변경한다.
- 팔레트에서 새 아이콘을 영역에 놓는 동작에만 부모 영역 자동 확장을 적용한다.
- 선택되지 않아 흐려진 다른 노드도 계속 클릭·선택·이동할 수 있게 한다.
- Resource name과 File name은 입력 중 빈 문자열을 보존하고 validation 오류를 표시한다.
- Region, Availability Zone, VPC, Subnet, Security Group, Auto Scaling Group 등 기존 영역 리소스의 기본 생성 크기는 유지한다.
- Terraform `resourceName`, `terraformBlockType`, 참조 문자열, 배포 동작은 변경하지 않는다.

## 현재 원인

### DATA 배지

`DiagramNodeView`가 `terraformBlockType === "data"`인 모든 노드에 `resourceNodeBadge`를 렌더링한다. 따라서 AMI뿐 아니라 다른 data source에도 같은 배지가 나타난다.

### 점선 그리드 침범

일반 리소스의 `nodeShellResource`와 `resourceNodeIconFrame` 배경이 투명하다. workspace의 점선 그리드가 아이콘 프레임 뒤로 비쳐 박스 안을 침범한 것처럼 보인다.

### 소문자와 특이한 접미사

`getResourceNodeLabel`이 친화적인 `node.label`보다 Terraform `parameters.resourceName`을 우선한다. `resourceName`에는 고유성 보장을 위한 `_2`, `_copy`, `_copy_2` 같은 접미사가 정상적으로 붙을 수 있어 내부 식별 문자열이 보드 표시명으로 노출된다.

### Resource name이 main으로 복구되는 문제

`mergeNodeParameters`가 저장된 `resourceName`과 `fileName`을 `||`로 기본값과 병합한다. 사용자가 입력을 지우면 빈 문자열이 falsy로 평가되어 다음 렌더에서 `main` 기본값으로 즉시 치환된다. 빈 값은 편집 중 유효한 중간 상태이며, 기본값은 필드가 실제로 없는 초기 생성에만 사용해야 한다.

## 설계

### 일반 리소스 생성 크기

모든 일반 아이콘 리소스의 기본 크기를 `48 × 48` 정사각형으로 통일한다. 실제 workspace catalog, 기본 palette, AI fallback, Terraform sync fallback이 같은 기본값을 사용한다. 일반 리소스의 resize 최소값은 `28 × 28`로 유지해 사용자가 더 작게 조절할 수 있게 한다.

`DiagramNode.size`는 아이콘 선택 영역만 뜻한다. 아이콘 프레임은 이 영역을 가득 사용하며 label 공간을 내부에서 예약하지 않는다.

영역 리소스 크기는 기존 값을 유지한다. 영역 리소스까지 절반으로 줄이면 헤더와 중첩 경계가 겹치고 자동 확장의 기준이 불안정해지기 때문이다.

### 아이콘 프레임과 선택 핸들

- `resourceNodeIconFrame`에 workspace와 같은 불투명한 흰색 배경을 적용한다.
- 선택 outline, 연결점, resize 핸들은 `48 × 48` 아이콘 영역만 감싼다.
- resource label은 아이콘 영역 아래 `4px` 위치에 absolute 배치하고 node height 계산에서 제외한다.
- 긴 label은 아이콘 아래에서 줄바꿈하며 아이콘 이미지나 resize 영역을 침범하지 않는다.
- 프레임의 이미지 비율과 선택 outline은 유지해 모든 AWS 아이콘에 같은 규칙을 적용한다.
- 수동 resize 핸들은 원형 채움에서 작은 정사각형으로 바꾼다.
- 핸들 내부는 투명하게 두고 얇은 보라색 테두리만 표시한다.
- hover와 focus에서는 크기를 키우지 않고 얇은 focus ring만 추가한다.

### 표시명

일반 리소스의 보드 표시명은 다음 우선순위를 사용한다.

1. 사용자가 명시한 `parameters.values.diagramLabel`
2. 리소스의 친화적인 `node.label`
3. 둘 다 비어 있을 때만 `node.type`

선택된 값은 trim한 뒤 대문자로 표시한다. Terraform `parameters.resourceName`은 표시명 후보에서 제외한다. 따라서 `ami_2`나 `bucket_copy`는 Terraform 내부에서 고유성을 유지하지만 보드에는 `AMI`, `S3 BUCKET` 같은 친화적인 대문자 이름만 보인다.

`DATA` 배지는 컴포넌트와 CSS에서 제거한다. 이는 시각 표시만 없애며 data source의 Terraform 의미는 유지한다.

### Metadata 입력

`mergeNodeParameters`는 `resourceName`과 `fileName`이 `undefined`일 때만 기본값을 사용한다. 저장된 값이 빈 문자열이면 그대로 입력에 전달해 사용자가 기존 이름 전체를 지운 뒤 새 이름을 입력할 수 있게 한다. 빈 문자열 상태에서는 기존 validation이 필수값 오류를 표시하며, 자동으로 `main`을 다시 채우지 않는다.

### 선택 상태 상호작용

하나 이상의 노드가 선택되면 선택되지 않은 노드는 기존 opacity 기반 dim 효과를 유지한다. dim은 시각 상태일 뿐 상호작용 상태가 아니다.

- 일반 노드는 dim 상태에서도 `selectable`, `draggable`, pointer interaction을 유지한다.
- 다른 일반 노드를 클릭하면 즉시 해당 노드로 단일 선택이 전환된다.
- 영역 노드는 기존 area blank-space hit-test를 통해 계속 선택할 수 있다.
- preview 상태처럼 본래 상호작용이 차단되는 경우는 이 규칙의 대상이 아니다.

### 부모 영역 자동 확장

팔레트에서 새 일반 아이콘을 영역에 drop하고 부모 지정이 끝난 순간에만 자동 확장 함수를 실행한다. 기존 노드를 같은 영역 안에서 움직이거나 다른 위치로 drag하는 동작은 자동 확장을 호출하지 않는다.

1. 새로 생성된 노드가 영역이 아닌 일반 아이콘이고 `parentAreaNodeId`가 있을 때만 처리한다.
2. 아이콘의 `parentAreaNodeId`에서 시작해 부모 체인을 가장 안쪽에서 바깥쪽으로 순회한다.
3. 각 부모 영역은 새 아이콘 너비와 높이의 두 배만큼 커진다.
4. 기본 `48 × 48` 아이콘이면 각 부모는 width와 height가 각각 `96px` 증가한다.
5. 부모 중심을 유지하기 위해 position은 x와 y에서 각각 `48px` 감소하고, 오른쪽과 아래쪽 경계는 각각 `48px` 늘어난다.
6. 모든 부모가 같은 양으로 상하좌우 확장되므로 기존 중첩 포함 관계를 유지한다.
7. 누락된 부모, 순환 참조, 영역이 아닌 부모를 만나면 안전하게 순회를 중단한다.

확장은 신규 drop 이벤트당 부모별 한 번만 실행한다. 자식의 위치, 기존 형제의 위치, Terraform metadata는 바꾸지 않는다.

## 구현 경계

- 표시명 계산은 렌더링 컴포넌트 밖의 순수 함수로 분리해 독립 테스트한다.
- 영역 확장은 `DiagramNode[]`, 새 노드 ID를 받아 새 배열을 반환하는 순수 함수로 둔다.
- 신규 drop 경로만 영역 확장 함수를 호출하고 기존 노드 drag 완료 경로에서는 호출하지 않는다.
- label은 node geometry 밖에 렌더링하되 같은 node view 안에 두어 별도 그래프 노드를 만들지 않는다.
- Terraform 동기화, AI draft 변환, Reverse Engineering 계약은 변경하지 않는다. 이 경로에서 만들어진 노드도 공통 표시 함수를 거치므로 시각 이름 규칙은 동일하게 적용된다.

## 테스트

- data source가 더 이상 `DATA` 배지를 렌더링하지 않는지 확인한다.
- 모든 일반 catalog·fallback 생성 경로가 `48 × 48`인지 확인한다.
- 일반 리소스 resize 최소값이 새 기본 크기를 허용하는지 확인한다.
- 아이콘 프레임이 node 크기를 모두 사용하고 label이 아래 외부에 배치되는지 확인한다.
- 표시명이 `diagramLabel`, `node.label`, `node.type` 순서로 선택되고 대문자로 변환되는지 확인한다.
- `resourceName`의 `_2`, `_copy`, 내부 ID가 표시명에 나타나지 않는지 확인한다.
- 새 `48 × 48` 아이콘 drop 시 각 부모가 중심을 유지하며 `96 × 96`만큼 커지는지 확인한다.
- 중첩된 부모 체인이 안쪽부터 바깥쪽까지 부모별 한 번씩 확장되는지 확인한다.
- 기존 아이콘을 영역 내부에서 움직이거나 다른 영역으로 drag할 때 크기가 바뀌지 않는지 확인한다.
- 부모 순환이나 누락된 부모가 있어도 무한 순회나 예외가 발생하지 않는지 확인한다.
- 빈 Resource name과 File name이 `main`으로 복구되지 않고 validation 오류 상태로 유지되는지 확인한다.
- dimmed 일반 노드를 클릭하면 선택이 전환되고 pointer interaction이 유지되는지 확인한다.
- CSS 소스 테스트와 브라우저 확인으로 아이콘 프레임 배경 및 resize 핸들 모양을 검증한다.

## 완료 기준

- workspace에서 AMI를 포함한 모든 data source 아래에 `DATA`가 보이지 않는다.
- 새 일반 아이콘 리소스가 `48 × 48` 정사각형으로 생성된다.
- 점선 그리드가 어떤 일반 리소스 아이콘 프레임 안에도 비치지 않는다.
- 선택 핸들이 작고 속이 빈 얇은 정사각형으로 보인다.
- 선택·연결·resize 영역은 아이콘만 감싸고 label은 아이콘 아래에서 가려지지 않는다.
- 신규 일반 아이콘을 영역에 넣으면 모든 부모 영역이 각각 아이콘 크기의 두 배만큼 중심 확장된다.
- 기존 아이콘을 이동해도 영역 크기는 변하지 않는다.
- 보드 리소스 이름은 대문자로 보이며 Terraform 고유성 접미사나 내부 ID가 노출되지 않는다.
- Resource name 전체를 지운 뒤 새 이름을 끊김 없이 입력할 수 있다.
- 선택되지 않아 흐려진 다른 노드도 계속 클릭·선택·이동할 수 있다.
- workspace 관련 단위 테스트, lint, typecheck, build가 통과한다.
