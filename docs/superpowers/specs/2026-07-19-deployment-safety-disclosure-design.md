# 배포 안전성 검사 결과 접기·펼치기 설계

## 목적

배포 안전성 검사 결과가 긴 Finding 목록으로 배포 화면을 과도하게 차지하지 않도록, 사용자가 필요할 때만 상세 내용을 펼쳐 확인할 수 있게 한다.

## 사용자 경험

- `HIGH`, `MEDIUM`, `LOW` 상태 배지와 `배포 안전성 검사 결과` 제목은 항상 표시한다.
- 제목 행 전체를 접기·펼치기 컨트롤로 사용한다.
- 최초 렌더링에서는 상세 내용을 접어 둔다.
- 제목 행을 클릭하거나 키보드로 활성화하면 다음 내용을 함께 펼친다.
  - 검사 결과 요약
  - Trivy 심층검사 상태
  - 발견 항목·실패·주의 통계
  - Finding 목록 또는 빈 결과 안내
- 제목 행 오른쪽에는 현재 상태를 나타내는 chevron을 표시하고, 펼쳐지면 방향을 회전한다.

## 구현 방식

`DeploymentPreDeploymentSummary`의 최상위 구조를 네이티브 `<details>`와 `<summary>`로 구성한다. `<summary>`에는 기존 상태 배지와 제목을 유지하고, 나머지 내용은 별도 상세 본문 컨테이너에 둔다. `open` 속성을 지정하지 않아 기본 상태를 접힘으로 유지한다.

별도의 React 상태나 검사 데이터 변환은 추가하지 않는다. 현재 `AiPreDeploymentAnalysisResult` 계산, Finding 동작, Terraform 소스 이동, AI 설명 요청 흐름은 그대로 유지한다.

## 시각·접근성

- 기존 배지, 카드, 본문, 통계, Finding 스타일을 유지한다.
- `<summary>`의 네이티브 키보드 조작을 사용하고 명확한 `:focus-visible` 표시를 제공한다.
- hover는 클릭 가능한 제목 행에만 적용한다.
- chevron 전환은 짧은 상태 변화로 제한하고 `prefers-reduced-motion`에서는 애니메이션을 제거한다.
- 모바일에서도 제목과 chevron이 겹치지 않고, 기존 통계 반응형 동작을 보존한다.

## 오류와 경계

- Trivy 실행·실패 메시지는 기존 조건을 그대로 사용하며, 접힌 동안에도 데이터 상태는 변경하지 않는다.
- Finding이 없을 때의 안내 문구도 상세 본문 안에서 기존처럼 렌더링한다.
- 접기·펼치기는 표현 상태만 바꾸며 배포 승인, Plan, Apply, API 호출에는 영향을 주지 않는다.

## 검증

- 정적 또는 컴포넌트 테스트로 `<details>`가 기본적으로 닫혀 있고 제목이 `<summary>`에 포함되는지 확인한다.
- 기존 배포 안전성 검사 데이터와 Finding 동작이 상세 본문 안에 그대로 유지되는지 확인한다.
- Web lint, typecheck, build와 저장소 필수 검사를 실행한다.
