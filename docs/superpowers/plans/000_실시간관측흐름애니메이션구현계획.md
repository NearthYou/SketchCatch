# 실시간 관측 흐름 애니메이션 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 실제 요청에만 트래픽 공을 표시하고, 유휴 연결선과 ECS Fargate Task Scale-out/Scale-in을 명확히 애니메이션하며, 불필요한 Architecture·설계 분석·하단 공백을 제거한다.

**Architecture:** 기존 `LiveObservationFocusedFlow`의 Architecture 기반 직선 경로를 유지한다. Provider snapshot은 표시할 실제·희망 Task 수를 결정하고, 별도의 순수 전환 helper가 사라지는 Task를 제한 시간 동안 보존해 퇴장 애니메이션을 가능하게 한다. 모달은 Focused Flow와 실제 관측 근거만 조합한다.

**Tech Stack:** React 19, TypeScript, CSS Modules, Node test runner, Next.js 16

## Global Constraints

- 유휴 상태에는 낮은 대비의 연결선 흐름만 표시하고 실제 요청 공이나 요청 수를 만들지 않는다.
- 실제 요청 증가에서만 한 burst당 최대 5개의 공을 표시한다.
- Fargate Task는 Architecture 최소 수와 Provider `running`/`desired`만 표시하고 `max`만으로 가상 슬롯을 만들지 않는다.
- `prefers-reduced-motion: reduce`에서는 반복 이동, 공 이동, 확대·축소를 중지하되 상태 텍스트와 Task 수는 유지한다.
- `전체 Architecture 보기`와 `WorkspaceDesignAnalysisPanel`을 Live Observation에서 제거한다.
- API, SSE, Redis, AWS SDK, Terraform, 부하 생성 코드는 변경하지 않는다.
- 새로운 runtime dependency와 DB migration을 추가하지 않는다.

- [ ] **Step 3: 전환 helper 최소 구현**

```ts
import type { LiveObservationCapacityUnit } from "./live-observation-diagram.js";

export const LIVE_OBSERVATION_CAPACITY_EXIT_MS = 620;

export type LiveObservationPresentedCapacityUnit = LiveObservationCapacityUnit & {
  readonly transition: "stable" | "exiting";
};

export function settleLiveObservationCapacityUnits(
  units: readonly LiveObservationCapacityUnit[]
): LiveObservationPresentedCapacityUnit[] {
  return units.map((unit) => ({ ...unit, transition: "stable" }));
}

export function reconcileLiveObservationCapacityUnits(
  current: readonly LiveObservationPresentedCapacityUnit[],
  next: readonly LiveObservationCapacityUnit[]
): LiveObservationPresentedCapacityUnit[] {
  const nextIds = new Set(next.map((unit) => unit.node.id));
  const exiting = current
    .filter((unit) => unit.transition === "stable" && !nextIds.has(unit.node.id))
    .map((unit) => ({ ...unit, transition: "exiting" as const }));
  return [...settleLiveObservationCapacityUnits(next), ...exiting];
}
```

- [ ] **Step 4: Focused Flow에 제한 시간 전환 상태 연결**

`LiveObservationFocusedFlow`에서 현재 모델 Task 목록을 상태로 보존한다.

```ts
const modelCapacityUnits = model.status === "ready" ? model.capacityUnits : [];
const [presentedCapacityUnits, setPresentedCapacityUnits] = useState(() =>
  settleLiveObservationCapacityUnits(modelCapacityUnits)
);

useEffect(() => {
  setPresentedCapacityUnits((current) =>
    reconcileLiveObservationCapacityUnits(current, modelCapacityUnits)
  );
  const timer = window.setTimeout(
    () => setPresentedCapacityUnits(settleLiveObservationCapacityUnits(modelCapacityUnits)),
    LIVE_OBSERVATION_CAPACITY_EXIT_MS
  );
  return () => window.clearTimeout(timer);
}, [modelCapacityUnits]);
```

렌더링은 `model.capacityUnits` 대신 `presentedCapacityUnits`를 사용하고 각 Task에 `data-transition={unit.transition}`을 추가한다.
