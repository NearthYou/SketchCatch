# Git Flow 예제

이 문서는 GitHub Issue, 작업 브랜치, Pull Request 연결 흐름을 확인하기 위한 예제입니다.

## 예제 흐름

1. Issue를 생성합니다.
2. Issue의 Development 영역에서 `dev` 기준 브랜치를 생성합니다.
3. 생성된 브랜치에서 작업합니다.
4. Pull Request를 `dev` 대상으로 생성합니다.
5. PR 본문에 `Closes #이슈번호`를 넣어 Issue와 PR을 연결합니다.

## 이번 예제

```text
Issue: #2
Branch: chore/codex/2-git-flow-example
Base branch: dev
```

이 파일은 실제 제품 기능이 아니라 브랜치/PR 운영 흐름 확인용입니다.
