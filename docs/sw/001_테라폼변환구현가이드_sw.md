# Terraform 변환 클론 코딩 가이드

이 문서는 SW Terraform 변환 기능을 사람이 그대로 따라 만들 수 있게 정리한 구현 가이드다. GitHub 이슈 본문에는 이 문서 내용을 길게 넣지 않고, 이슈에서는 `docs/sw/spec.md`와 `docs/sw/plan.md`만 참조한다.

코드 블록에 `...`가 있으면 혼자 따라 치기 어렵기 때문에, 핵심 파일은 전체 코드를 그대로 적는다.

## 1. 만들 순서

1. `packages/types`에 `DiagramJson` 공유 타입을 추가한다.
2. `apps/api`에 순수 변환 함수와 테스트를 만든다.
3. API route를 붙여 웹에서 변환기를 호출할 수 있게 한다.
4. `apps/web` workspace에 샘플 DiagramJson과 Terraform preview UI를 연결한다.
5. 정적 diagnostics와 코드 to DiagramJson 동기화는 후속 이슈에서 확장한다.

## 2. 생성할 폴더와 파일

```text
packages/types/src/index.ts
apps/api/src/services/terraform/diagramToTerraform.ts
apps/api/src/services/terraform/diagramToTerraform.test.ts
apps/api/src/routes/terraform.ts
apps/api/src/routes/terraform.test.ts
apps/web/app/workspace/AiWorkspaceClient.tsx
```

`apps/api/src/services/terraform` 폴더가 없으면 새로 만든다. 첫 이슈에서는 DB, S3, Terraform CLI, AWS SDK를 호출하는 파일을 만들지 않는다.

## 3. 공유 타입 추가

## 4. 순수 변환 함수 구현

`apps/api/src/services/terraform/diagramToTerraform.ts` 파일을 만들고 아래 코드를 그대로 작성한다. helper를 나중에 상상해서 추가하는 방식이 아니라, 처음부터 전체 helper를 포함해서 만든다.

```ts
import type {
  DiagramJson,
  DiagramNodeParameters,
  TerraformBlockType
} from "@sketchcatch/types";

const DEFAULT_TERRAFORM_BLOCK_TYPE: TerraformBlockType = "resource";
const INDENT_UNIT = "  ";
const TERRAFORM_REFERENCE_PATTERN =
  /^[a-zA-Z_][a-zA-Z0-9_]*\.[a-zA-Z_][a-zA-Z0-9_]*\.[a-zA-Z_][a-zA-Z0-9_]*(?:\.[a-zA-Z_][a-zA-Z0-9_]*)*$/;

// DiagramJson 전체를 Terraform 코드 문자열 하나로 변환하는 공개 함수다.
// DB, S3, Terraform CLI, AWS SDK를 호출하지 않는 순수 함수로 유지한다.
export function generateTerraformFromDiagramJson(diagramJson: DiagramJson): string {
  return diagramJson.nodes
    .filter((node) => node.kind === "resource")
    .map((node) => node.parameters)
    .filter(isRenderableParameters)
    .map(renderBlock)
    .join("\n\n");
}

// Terraform으로 출력해도 되는 node parameters인지 검사한다.
// parameters가 없거나 invalid 표시가 있는 resource node는 조용히 제외한다.
function isRenderableParameters(
  parameters: DiagramNodeParameters | undefined
): parameters is DiagramNodeParameters {
  return parameters !== undefined && parameters.invalid !== true;
}

// resource/data block 하나를 만든다.
// 예: resource "aws_vpc" "main" { ... }
function renderBlock(parameters: DiagramNodeParameters): string {
  const terraformBlockType =
    parameters.terraformBlockType ?? DEFAULT_TERRAFORM_BLOCK_TYPE;
  const body = Object.entries(parameters.values).map(([key, value]) =>
    renderAttribute(key, value, 1)
  );

  return [
    `${terraformBlockType} "${parameters.resourceType}" "${parameters.resourceName}" {`,
    ...body,
    "}"
  ].join("\n");
}

// values의 key/value 하나를 Terraform attribute 한 줄로 바꾼다.
// 예: enableDnsSupport: true -> enable_dns_support = true
function renderAttribute(key: string, value: unknown, indentLevel: number): string {
  return `${indent(indentLevel)}${toSnakeCase(key)} = ${renderValue(value, indentLevel)}`;
}

// JavaScript 값을 Terraform HCL 값 표현으로 바꾼다.
// string, number, boolean, null, object, array를 모두 이 함수에서 분기한다.
function renderValue(value: unknown, indentLevel: number): string {
  if (value === null || value === undefined) {
    return "null";
  }

  if (typeof value === "string") {
    return isTerraformReference(value) ? value : JSON.stringify(value);
  }

  if (typeof value === "number" || typeof value === "boolean") {
    return String(value);
  }

  if (Array.isArray(value)) {
    return renderArray(value, indentLevel);
  }

  if (isRecord(value)) {
    return renderObject(value, indentLevel);
  }

  return JSON.stringify(String(value));
}

// 배열 값을 Terraform list 표현으로 바꾼다.
// 여러 줄로 출력해서 사람이 읽고 수정하기 쉽게 만든다.
function renderArray(values: unknown[], indentLevel: number): string {
  if (values.length === 0) {
    return "[]";
  }

  return [
    "[",
    ...values.map((value) => `${indent(indentLevel + 1)}${renderValue(value, indentLevel + 1)},`),
    `${indent(indentLevel)}]`
  ].join("\n");
}

// object 값을 Terraform map/object 표현으로 바꾼다.
// tags = { Name = "main-vpc" } 같은 중첩 값을 담당한다.
function renderObject(value: Record<string, unknown>, indentLevel: number): string {
  const entries = Object.entries(value);

  if (entries.length === 0) {
    return "{}";
  }

  return [
    "{",
    ...entries.map(
      ([key, nestedValue]) =>
        `${indent(indentLevel + 1)}${key} = ${renderValue(nestedValue, indentLevel + 1)}`
    ),
    `${indent(indentLevel)}}`
  ].join("\n");
}

// null과 array를 제외한 일반 object인지 확인한다.
// renderValue에서 object branch를 안전하게 태우기 위한 타입 가드다.
function isRecord(value: unknown): value is Record<string, unknown> {
  return typeof value === "object" && value !== null;
}

// "aws_vpc.main.id" 같은 Terraform reference 문자열인지 판별한다.
// reference는 따옴표를 붙이면 안 되므로 일반 문자열과 구분해야 한다.
function isTerraformReference(value: string): boolean {
  return TERRAFORM_REFERENCE_PATTERN.test(value);
}

// DiagramJson의 camelCase key를 Terraform attribute naming인 snake_case로 바꾼다.
// 예: mapPublicIpOnLaunch -> map_public_ip_on_launch
function toSnakeCase(value: string): string {
  return value
    .replace(/([A-Z]+)([A-Z][a-z])/g, "$1_$2")
    .replace(/([a-z0-9])([A-Z])/g, "$1_$2")
    .replace(/-/g, "_")
    .toLowerCase();
}

// indent level을 공백 문자열로 바꾼다.
// Terraform 예제는 2칸 들여쓰기를 기본으로 맞춘다.
function indent(level: number): string {
  return INDENT_UNIT.repeat(level);
}
```

첫 구현은 입력 node 순서를 유지한다. `fileName`별 분리는 후속 export 이슈에서 처리한다.

## 6. API route 연결

순수 변환기 테스트가 통과한 뒤 `apps/api/src/routes/terraform.ts`를 만든다. 현재 제품 방향은 로그인한 회원의 workspace만 허용하므로, DB를 읽지 않는 API라도 `requireActiveUserId`를 호출한다.

```ts
import type { FastifyInstance } from "fastify";
import { z } from "zod";
import type { DiagramJson } from "@sketchcatch/types";
import { requireActiveUserId } from "../auth/current-user.js";
import { getDatabaseClient, type DatabaseClient } from "../db/client.js";
import { generateTerraformFromDiagramJson } from "../services/terraform/diagramToTerraform.js";

const terraformBlockTypeSchema = z.enum(["resource", "data"]);

const diagramNodeParametersSchema = z.object({
  terraformBlockType: terraformBlockTypeSchema.optional(),
  resourceType: z.string().min(1),
  resourceName: z.string().min(1),
  fileName: z.string().min(1),
  values: z.record(z.string(), z.unknown()),
  invalid: z.boolean().optional()
});

const diagramNodeSchema = z.object({
  id: z.string().min(1),
  type: z.string().min(1),
  kind: z.enum(["resource", "design"]),
  label: z.string().min(1),
  parameters: diagramNodeParametersSchema.optional()
});

const diagramEdgeSchema = z.object({
  id: z.string().min(1),
  sourceNodeId: z.string().min(1),
  targetNodeId: z.string().min(1),
  label: z.string().min(1).optional()
});

const diagramJsonSchema: z.ZodType<DiagramJson> = z.object({
  nodes: z.array(diagramNodeSchema),
  edges: z.array(diagramEdgeSchema),
  viewport: z.object({
    x: z.number(),
    y: z.number(),
    zoom: z.number()
  })
});

const terraformGenerateBodySchema = z.object({
  diagramJson: diagramJsonSchema
});

type TerraformRouteOptions = {
  getDatabaseClient?: () => DatabaseClient;
};

// Terraform 관련 API route를 Fastify app에 등록한다.
// app.ts에서 prefix: "/api"로 등록하므로 실제 경로는 /api/terraform/generate가 된다.
export async function registerTerraformRoutes(
  app: FastifyInstance,
  options: TerraformRouteOptions = {}
): Promise<void> {
  const getTerraformDatabaseClient = options.getDatabaseClient ?? getDatabaseClient;

  // DiagramJson을 요청 body로 받아 Terraform 코드 문자열을 반환한다.
  // 저장된 DB 데이터를 읽지 않지만, 로그인 회원 workspace 정책을 맞추기 위해 인증은 검사한다.
  app.post("/terraform/generate", async (request) => {
    await requireActiveUserId(request, getTerraformDatabaseClient);

    const body = terraformGenerateBodySchema.parse(request.body);

    return {
      terraformCode: generateTerraformFromDiagramJson(body.diagramJson)
    };
  });
}
```

```ts
// 테스트 요청에 넣을 Authorization header를 만든다.
// 실제 서비스와 같은 access token 생성 helper를 사용한다.
function authHeaders(userId: string): Record<string, string> {
  return {
    authorization: `Bearer ${createAccessToken(userId)}`
  };
}

// API 에러 응답이 공통 에러 DTO 형식을 지키는지 확인한다.
function assertErrorResponse(response: ApiErrorResponse, error: ApiErrorResponse["error"]): void {
  assert.equal(response.error, error);
  assert.equal(typeof response.message, "string");
  assert.ok(response.message.length > 0);
}

// 실제 DB를 띄우지 않고 users 조회 결과만 흉내 낸다.
class AuthOnlyFakeDb {
  readonly client: DatabaseClient;
  private readonly users: AuthOnlyFakeUser[];

  constructor(options: { users: AuthOnlyFakeUser[] }) {
    this.users = options.users;
    this.client = {
      db: this as unknown as DatabaseClient["db"],
      pool: {
        end: async () => undefined
      } as DatabaseClient["pool"]
    };
  }

  // requireActiveUserId 내부의 db.select().from().where() 호출 체인만 지원한다.
  select(): {
    from: () => {
      where: () => Promise<AuthOnlyFakeUser[]>;
    };
  } {
    return {
      from: () => ({
        where: async () => this.users
      })
    };
  }
}
```

## 7. 웹 연결

`apps/web/app/workspace/AiWorkspaceClient.tsx`에는 처음부터 DB 연동을 넣지 않는다.

1. 파일 안에 샘플 `DiagramJson`을 둔다.
2. `apiFetch`로 `POST /api/terraform/generate`를 호출한다.
3. 응답의 `terraformCode`를 textarea에 표시한다.
4. loading, error, empty 상태를 기존 UI 패턴에 맞춰 보여준다.

호출 helper는 아래처럼 시작한다.

```ts
import type { DiagramJson } from "@sketchcatch/types";
import { apiFetch } from "../../lib/api-client";

type TerraformGenerateResponse = {
  terraformCode: string;
};

// workspace 화면에서 DiagramJson을 API로 보내고 Terraform 코드만 꺼내오는 helper다.
// auth: true를 사용해서 로그인 세션의 access token이 Authorization header에 들어가게 한다.
async function generateTerraform(diagramJson: DiagramJson): Promise<string> {
  const response = await apiFetch<TerraformGenerateResponse>("/terraform/generate", {
    method: "POST",
    auth: true,
    body: {
      diagramJson
    }
  });

  return response.terraformCode;
}
```

프론트엔드는 Terraform CLI, AWS SDK, S3 upload, RDS persistence를 직접 실행하지 않는다.
