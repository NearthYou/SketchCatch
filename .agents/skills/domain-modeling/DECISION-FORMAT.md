# Architecture Decision Format

Architecture decisions live in the `기술 결정 기록` section of `docs/architecture.md`. Do not create a separate file for each decision.

## Template

```md
### 결정 {N}: {Short title}

{1-3 sentences explaining the context, decision, and reason.}
```

Add considered options or consequences only when they prevent a likely future mistake. Scan existing decision headings for the next number, and update other canonical documents when the decision changes product scope, data contracts, or deployment behavior.
